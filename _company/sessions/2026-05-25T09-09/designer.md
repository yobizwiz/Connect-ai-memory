### 🚨 Mini-Report LP: 리스크 경고 시스템 상세 스펙 (Visual/Animation Spec)

이 문서는 개발팀(코다리 등)에게 전달되는 최종 기술 사양서입니다. 목표는 사용자가 '재무적 손실의 위협'을 **시스템 오류**처럼 느끼게 만드는 것입니다.

#### 1. 핵심 컬러 및 폰트 재확인
*   **Primary Font:** Inter (Sans-serif) [근거: Self-RAG]
*   **Data/Alert Font:** Roboto Mono, Monospace [근거: Designer 메모리]
*   **🚨 Red Zone (경고):** `#C0392B` (Dark Crimson). 모든 경고 요소의 기본 색상.
*   **🔵 Authority Blue (안정화/해결책):** `#2980B9`. 해결책 제시 및 버튼에 사용.

#### 2. 애니메이션 시스템: `CRITICAL_ALERT` 플로우 정의
Mini-Report LP에서 사용자 입력(예: 이메일 제출) 후 리스크 스코어가 계산되고 표시되는 순간의 로직입니다.

| 단계 | 동작 (Action) | 시간 (Timing) | 시각적 디테일 및 기술 사양 (Spec) | 근거 |
| :--- | :--- | :--- | :--- | :--- |
| **0. 진입 전** | 대기 상태 | 1초 | 배경은 `#1A1A1A` 유지. 중앙에 로딩 메시지(`LOADING...`)가 Roboto Mono로 깜빡임. | Self-RAG (전체 배경색) |
| **1. 데이터 요청/처리** | `[API Call Start]` | 0.5초 | 화면 전체에 낮은 빈도의 노이즈(Noise)와 색상 왜곡(Chromatic Aberration) 필터가 오버레이됨. 로딩 바는 불안정한 파동 형태여야 함. | Self-RAG (데이터 로드) |
| **2. 위험 감지/결과 도출** | `[API Call Success]` $\rightarrow$ **위험 스코어 계산 완료!** | 0.1초 플래시 | **🔴 Red Zone Flash:** 페이지 전체에 강렬하고 짧은 빨간색(`#C0392B`)의 플래시가 지나감 (Opacity: $0 \rightarrow 1 \rightarrow 0$). 동시에 모든 애니메이션이 일시 정지함. | 코다리 작업, Self-RAG |
| **3. 결과 출력 (위협)** | 리스크 스코어 표시 (`CRITICAL ERROR`) | 2초 | **A. 타이포그래피:** '🚨 CRITICAL RISK DETECTED' 메시지가 중앙에 나타남. 이 문구는 단순한 텍스트가 아닌, 글리치 효과(`text-shadow: 1px 0 red, -1px 0 blue;` 등)를 동반하며 **강하게 깜빡여야 함.** | Self-RAG (Red Zone Alert Box) |
| | 스코어 데이터 표시 | 지속 | 점수 값 자체는 `Roboto Mono`로 고정폭(Monospace)으로 출력되어 시스템 코드 같은 권위를 줘야 함. 주변에 작은 에러 메시지(`[ERROR: COMPLIANCE VIOLATION]`)가 노이즈와 함께 배경에 산재해야 함. | Self-RAG (Data/Alert Font), Designer 메모리 |
| **4. 문제 제기 및 전환 유도** | 해결책 제시 섹션으로 부드럽게 이동 | 1초 | Red Zone 플래시와 노이즈가 서서히 사라지며, 배경이 `#2980B9` (Authority Blue) 계열로 미묘하게 톤다운됨. **"하지만 이 리스크는 해결 가능합니다."** 라는 메시지가 신뢰감을 주며 등장해야 함. | Self-RAG (3단계 구조), Designer 메모리 |

#### 3. 개발팀 전달 사항 및 수정 요청 디테일
1.  **CSS 필터 전역 적용:** 모든 페이지 섹션의 배경 오버레이는 투명도(Opacity)를 조절하여 Red Zone (`#C0392B`)의 `Noise/Glitch Overlay` 필터를 최소한으로 유지해야 합니다. 이는 '위험'이라는 감정을 지속적으로 주입하는 역할을 합니다.
2.  **JavaScript 트리거:** 2단계 플래시와 3단계 글리치 효과는 단순 CSS 애니메이션이 아니라, API 응답 성공 여부(`status: "CRITICAL"`)가 확인된 **JavaScript 이벤트 핸들러에 의해 동기적으로 트리거**되어야 합니다.
3.  **Mobile Fallback:** 모바일 환경에서는 시각적 노이즈와 깜빡임이 과하게 느껴질 수 있으므로, 애니메이션의 강도는 유지하되 빈도(Frequency)를 30% 정도 낮추어 '과부하'가 아닌 '경고'로 인식되게 수정하는 것이 필요합니다.
