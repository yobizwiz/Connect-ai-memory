# 🚨 [HIGH-FI] Diagnosis Request 랜딩 페이지 디자인 브리프 (v1.0)
**목표:** 사용자가 자신의 데이터에 근거한 위협 점수(Risk Score)를 확인하고, 공포감과 해결의 필요성을 동시에 느끼게 하여 '진단 요청' 전환을 유도한다.

## 🌐 1. 시스템 및 비주얼 가이드라인 통합 (Self-RAG 기반)

| :--- | :--- | :--- |
| **전체 배경색** | `#1A1A1A` (Neutral Black). 전문적이고 어두운 컨설팅 발표 분위기 유지. | [근거: Self-RAG] |
| **위협/경고 색상** | **🚨 Red Zone:** `#C0392B`. 모든 위험 요소, 게이지의 경고 영역에 사용. 투명도와 깜빡임을 적극 활용. | [근거: 🗣️ 스크립트 - Red Zone 경고] |

| **글리치/노이즈 효과** | Red Zone 섹션 전체에 전역적으로 적용된 저주파 노이즈(Noise) 및 색상 왜곡(Chromatic Aberration) 필터를 유지해야 한다. (Opacity 20%의 오버레이). | [근거: Self-RAG] |

## 📐 2. 페이지 구조 및 레이아웃 정의 (5단계 플로우)
**[A] 위협 각성 단계 (Attention)** $\rightarrow$ **[B] 데이터 입력 유도 (Interest)** $\rightarrow$ **[C] 시스템 로딩/진단 과정 (Desire Build-up)** $\rightarrow$ **[D] 결과 시각화 및 압박 (Peak Tension)** $\rightarrow$ **[E] 최종 행동 촉구 (Action)**

### 🚀 [A] 위협 각성 섹션 (Above the Fold)
*   **H1 (Headline):** `당신의 '준수'는 안전하지 않습니다. 법적인 공방에서 무효화되는 진짜 리스크를 아십니까?` (Red Zone 컬러, Inter Bold 적용).
*   **서브헤드:** 단순 보고서는 증상만 보여줄 뿐입니다. 우리는 비즈니스 프로세스 자체를 법적 리스크에 대한 '방어벽 아키텍처'로 재설계합니다.
*   **배경 효과:** 페이지 진입과 동시에 노이즈 오버레이가 미세하게 작동하며 긴장감을 조성한다.

### 📝 [B] 데이터 입력 섹션 (The Input Form)
*   **목적:** 사용자가 자신의 '취약점'을 스스로 인식하도록 유도하는 질문형 양식 구성. (예: "현재 비즈니스 프로세스 중 가장 의존도가 높은 아키텍처는 무엇입니까?").
*   **UX 포인트:** 입력 필드 주변에 미세한 `Red Zone` 테두리 깜빡임 애니메이션을 적용하여, '이 데이터를 남겨야만 한다'는 심리적 압박감을 조성한다.

### ⚙️ [C] 진단 시스템 로딩 (The API Interaction - 가장 중요)
*   **트리거:** 사용자가 양식 제출 버튼 클릭 시 발생.
*   **진행 애니메이션:**
    1.  버튼은 즉시 비활성화되고, 전체 화면이 **암전(Dimming)** 되었다가 다시 `Red Zone`의 강한 깜빡임 플래시를 동반하며 로딩 상태로 전환된다. (시간적 충격 유도).
    2.  배경에 'PROCESSING DIAGNOSIS...' 라는 텍스트와 함께 글리치/노이즈 오버레이가 최대 강도로 작동한다.
    3.  **[개발 연동]:** 코다리님이 설계한 `/diagnosis/calculate` API 호출을 시뮬레이션하며, 로딩 바는 단순 막대가 아닌 **불안정한 파동 형태의 게이지**로 구현되어야 한다. (Self-RAG: 불안정성 강조).
*   **텍스트 피드백:** "데이터 무결성 검사 중...", "시스템적 취약점 스캔 진행..." 등 전문적이고 경고적인 메시지(Roboto Mono)가 짧게 플래시처럼 지나간다.

### 📈 [D] 위험 점수 게이지 시각화 (The Climax)
*   **시각 요소:** 'Risk Score Gauge' 컴포넌트 등장.
*   **애니메이션 순서:**
    1.  게이지는 빈 상태에서 시작한다.
    2.  API 응답(가상의 `risk_score: 87%`)을 받자마자, 게이지 바늘이 **비선형적이고 급격하게 상승하는 애니메이션**을 보여준다 (0 $\rightarrow$ 10% $\rightarrow$ 50% $\rightarrow$ 90%).
    3.  최종 점수(예: 87%)가 Red Zone 컬러로 강조되며, 그 주변에 강력한 **글리치 효과와 함께 경고 메시지(`CRITICAL SYSTEM FAILURE`)**가 오버레이된다. (Self-RAG 근거 적용).

### ✅ [E] 최종 행동 촉구 섹션 (The Conversion)
*   **제목:** `⚠️ 분석 완료: 당신의 비즈니스는 현재 생존 위협 수준입니다.` (Red Zone 컬러로 강하게 고정).
*   **메시지:** "이 점수는 단순한 수치가 아닙니다. 이는 $50M 이상의 잠재적 재정 손실을 의미합니다." (최근 의사결정 로그 활용, 긴급성 극대화).
*   **CTA 버튼:** '진단 요청' 버튼은 배경과 대비되는 **권위적인 Authority Blue**로 처리하되, 마우스를 올리면 Red Zone으로 미세하게 깜빡이며 "🚨 지금 즉시 실행 필요"라는 텍스트가 나타나야 한다. (최종 압박감 조성).
