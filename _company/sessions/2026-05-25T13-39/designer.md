## 🚨 Mini-Report LP: Pre-Purchase Error Mockup Spec (Red Zone Gatekeeping)

### I. 목표 및 사용자 경험 정의
*   **목표:** 사용자가 결제에 실패하는 순간, '결제 시스템 오류'라는 사소한 문제로 의식을 전환시키는 것이 아니라, **'자신의 비즈니스 구조적 취약점'이라는 더 큰 공포**에 몰입하게 만든다.
*   **전환 메커니즘:** 사용자가 $X$ 금액을 지불하려는 시도 $\rightarrow$ 시스템 차원의 강렬한 방해/경고(Red Zone) 발생 $\rightarrow$ 문제 해결을 위해서는 *전문가적 개입*이 필수라는 결론 도출 $\rightarrow$ 유료 진단 체험 CTA로 수렴.
*   **전체 분위기:** 전문 컨설팅의 민감 정보 접근 시도가 실패한 듯한, '시스템 레벨'의 위협과 권위적인 경고 톤앤매너 유지.

### II. 디자인 구성 요소 (Visual Components)

#### A. 전체 배경 및 애니메이션
1.  **배경색:** `#1A1A1A` (Neutral Black).
2.  **로딩/전환 효과:** 사용자가 결제 버튼을 클릭하는 순간, 페이지 전체에 **짧고 강렬한 노이즈(Noise) 오버레이**가 발생하며 화면이 멈춘다 (Stuttering effect).
3.  **핵심 애니메이션:** 모든 경고 메시지 및 데이터는 일반적인 글씨체/배치 대신, 시스템 오류창처럼 깜빡임 (`text-shadow: 1px 0 red, -1px 0 blue;`)과 글리치 효과를 적용해야 합니다.

### III. 기술적 구현 브리프 (Developer Notes)
1.  **[Error State Trigger]**: 결제 API 호출 실패 시(`status: failure`), 즉시 이 Red Zone View로 리디렉션되어야 합니다.
2.  **[Interactivity]**: 사용자가 스크롤을 하거나, 페이지에 머무르는 시간(Dwell Time)이 길수록 3단계의 `$50만~$8M` 경고 수치가 미세하게 깜빡이는 빈도가 높아져 긴장감을 유지해야 합니다.
3.  **[기술 구현]**: CSS 애니메이션 (`@keyframes`)과 JavaScript를 활용하여, '글리치 효과'와 '불규칙한 플래시/깜빡임'을 흉내 내는 것이 필수적입니다.
