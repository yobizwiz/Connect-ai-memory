<create_file path="~/Documents/yobizwiz/design_spec/gatekeeper_warning_v2.0.md"># 🚨 시스템 경고: 게이트키퍼(Gatekeeper) 애니메이션 상세 스펙 V2.0
> **목표:** 사용자에게 '정보 입력의 공백'이 곧 '법적 리스크'라는 존재론적 불안감을 극대화한다. 단순한 UI 오류가 아닌, *시스템 자체의 붕괴*를 체험하게 한다.

## ⚙️ 전역 애니메이션 원칙 (Global Animation Principles)
1. **폰트 계층 구조:**
    *   **H1/메인 경고 문구:** Inter (Sans-serif), 고대비의 두꺼운 볼드체로 공포를 전달한다.
    *   **경고 코드/데이터:** Roboto Mono, 모든 경고 메시지나 리스크 스코어는 이 폰트를 사용하며, 이는 '시스템 레벨 데이터'임을 각인시킨다.
2. **컬러 프로토콜 (Color Protocol):**
    *   `#1A1A1A` (Neutral Black) $\rightarrow$ 배경 기본값.
    *   `#C0392B` (Red Zone) $\rightarrow$ 경고 발생 시 주 색상. 모든 플래시, 오버레이, 강조 요소에 사용된다.
    *   `#2980B9` (Authority Blue) $\rightarrow$ 시스템 안정화/해결책 제시 부분에서만 일시적으로 사용되어 대비 효과를 극대화한다.
3. **공통 시각 효과:** 모든 경고 발생(Trigger) 순간에는 다음 두 가지 필터가 동시에 작동해야 합니다.
    *   **글리치 노이즈 (Glitch Noise):** 전체 화면에 낮은 빈도의 색상 채널 분리(Chromatic Aberration, R/G/B 채널의 미세한 어긋남)와 짧은 노이즈 오버레이를 적용합니다.
    *   **시스템 스터터링 (System Stuttering):** 애니메이션 프레임 간에 10~30ms 단위로 화면 전체가 순간적으로 떨리거나(Shake), 깜빡이는(Flicker) 효과를 주어 '불안정성'을 증명합니다.

## 💣 트리거별 인터랙티브 상세 스펙 (Trigger-Specific Specs)

### Case A: 정보 부재 경고 (The Void Alert - [A-Type])
*   **발생 상황:** 사용자가 필수 필드를 비워두거나, 데이터 형식이 잘못되었을 때. (위협 레벨: Low-Medium)
*   **목표 감정:** "어딘가 빠져있다"는 미묘한 불편함과 불완전성.

| 단계 | 시간/지속시간 | 시각적 동작 (Visual Action) | 기술 구현 지시 (Technical Specs) |
| :--- | :--- | :--- | :--- |
| **1. 감지** | 0ms - 200ms | 입력 필드 근처에 투명도가 낮은 `#C0392B`의 물결 노이즈가 확산된다. (Wave Propagation) | `CSS`: `:focus-within` 또는 `onBlur` 이벤트 발생 시, 해당 영역 주변에 가우시안 블러(Gaussian Blur)와 함께 Red Zone 색상 오버레이를 씌운다. |
| **2. 경고 노출** | 200ms - 500ms | [A-Type] 문구 출력. 일반 텍스트가 아닌, `Roboto Mono`의 코드로 출력되며 글리치 효과와 함께 깜빡인다. | `JS/CSS`: 텍스트에 짧은 간격으로 (150ms) Opacity 토글을 적용하여 '데이터 전송 오류' 같은 느낌을 준다. |
| **3. 강조** | 500ms - 700ms | 필드명(`[필드명]`)이 자동으로 빨간색으로 깜빡이며 커서가 그 위에 강제적으로 고정된다 (Cursor Trap). | `CSS`: 해당 요소에 `:active` 상태를 정의하고, 마우스 커서와 함께 Red Zone 색상의 미세한 떨림(Jitter) 효과를 부여한다. |

---

### Case C: 임계값 경보 (CRITICAL THRESHOLD WARNING - [C-Type])
*   **발생 상황:** 리스크 스코어가 70%에 도달했을 때. 시스템이 사용자에게 '경고' 메시지를 발송하는 느낌. (위협 레벨: High)
*   **목표 감정:** "시스템이 나를 보고있다"는 감시와 긴장감.

| 단계 | 시간/지속시간 | 시각적 동작 (Visual Action) | 기술 구현 지시 (Technical Specs) |
| :--- | :--- | :--- | :--- |
| **1. 트리거** | 0ms - 50ms | 화면 전체가 순간적으로 어둡게(Neutral Black $\rightarrow$ Opacity 0.8) 처리되고, 경고 발생 위치에 진동(Vibration)이 감지된다. | `JS`: 전역 애니메이션 클래스(`is-system-alert`)를 추가하고, 짧은 CSS `transform: translate()` 떨림을 적용한다. |
| **2. 스코어 제시** | 50ms - 300ms | [C-Type] 문구와 함께 '72%'라는 수치가 중앙에 고정된다. 이 숫자는 아날로그 계기판이 과부하되는 것처럼 시각화되어야 한다. | `CSS/SVG`: 숫자 주변에 Red Zone 색상의 파동(Ripple) 효과를 주어, 데이터가 임계점을 넘는 물리적 충격을 표현한다. |
| **3. 경고 증폭** | 300ms - 800ms | 배경 전체에 글리치 노이즈와 함께 `⚠️` 이모지가 깜빡이며, 페이지의 다른 요소들(버튼 등)까지 Red Zone 필터가 미세하게 침범한다 (Bleeding Effect). | `CSS Filter`: 전역적으로 `filter: hue-rotate(-30deg) brightness(1.1)` 같은 왜곡 효과를 주기적으로 적용하여 '시스템 불안정'을 증명한다. |

---

### Case D: 치명적 위험 경보 (CRITICAL FAILURE - [D-Type])
*   **발생 상황:** 리스크 스코어가 90%에 도달했을 때. 컴플라이언스 게이트웨이가 강제 종료되는 상황. (위협 레벨: CRITICAL)
*   **목표 감정:** "이대로는 안 된다"는 절대적 패닉과 행동 불가(Action Impossible).

| 단계 | 시간/지속시간 | 시각적 동작 (Visual Action) | 기술 구현 지시 (Technical Specs) |
| :--- | :--- | :--- | :--- |
| **1. 전면 차단** | 0ms - 200ms | 화면 전체가 `#C0392B`의 강렬한 플래시(Flash)로 완전히 뒤덮인다 (Opacity 0 $\rightarrow$ 1 $\rightarrow$ 0). 모든 입력을 일시적으로 비활성화한다. | `JS`: `<body>` 태그에 배경 색상 및 오버레이를 즉시 적용하고, 포커스 이벤트를 강제로 막는다 (`preventDefault`). |
| **2. 시스템 메시지** | 200ms - 500ms | [D-Type] 문구와 함께, `[SYSTEM OVERRIDE]`라는 헤드라인이 화면 중앙에 '강제 오버레이'된다. 이 문구는 글리치 효과가 가장 극대화되어야 한다. | `CSS`: `text-shadow: 1px 0 #C0392B, -1px 0 #000;` 같은 다중 그림자(Multi-Shadow)를 사용하여 디지털 노이즈의 깊이를 표현한다. |
| **3. 강제 고정** | 500ms - 1500ms | '법적 분쟁 발생 시 서비스 전체의 존재론적 위협' 문구가 화면 중앙에 *반복적으로* 깜빡이며 (Blink/Pulse), 주변 데이터 필드들은 마치 먹통이 된 것처럼 흐릿하게(Blur) 처리된다. | `JS`: 텍스트 내용 자체를 주기적으로 변경하거나 (예: "위협..." $\rightarrow$ "...위협"), Opacity와 Scale을 불규칙하게 변화시켜 시선을 강제한다. 이 상태에서는 *어떤 액션도 허용되지 않음*을 명확히 해야 한다. |

---
**디자인/개발 유의사항:**
1.  **사운드 디자인 통합 필수:** 경고 발생(Trigger) 시, 단순히 색상만 바뀌는 것이 아니라, 낮은 주파수의 **삐-익 (Beeper)** 소리와 함께 **데이터 손실음(Data Corruption Sound)**을 재생하여 청각적 긴급성을 극대화해야 합니다.
2.  **반복 방지:** 경고 애니메이션은 짧게(최대 1.5초) 끝나되, 그 여운이 길게 남아야 합니다. 경고가 끝난 후에도 화면 전체에 미세한 Red Zone의 잔상(Afterglow)을 유지하여 '위험했던 경험'을 기억하게 만들어야 합니다.
