## 🚨 [최종 산출물] Risk Quantification Module: 디자인 시스템 청사진 (Blueprint)
**목표:** 사용자가 '위험한 상태'임을 감정적 공포가 아닌, '객관적인 수치와 구조적 결함'으로 체감하게 한다.
**활용 위치:** 랜딩 페이지의 핵심 섹션 ('⚠️ 경고: 당신이 받은 보고서가 알려주지 않는 것') 및 최종 세일즈 피치 덱 전반.

### 1. 컬러 팔레트 & 타이포그래피 재확인 (Foundation)
| 역할 | 색상명/코드 | 사용 목적 및 효과 | 근거 |
| :--- | :--- | :--- | :--- |
| **🚨 Red Zone** | `#C0392B` (Dark Crimson) | 위험, 경고, 임계치 초과. 글리치와 결합하여 시각적 충격 극대화. | Self-RAG 메모리 |
| **🔵 Authority Blue** | `#2980B9` (Deep Slate Blue) | 해결책, 안정성, 데이터의 출처. 시스템의 신뢰 영역. | Self-RAG 메모리 |
| **⚫ Neutral Black** | `#1A1A1A` | 기본 배경색. 전문적이고 깊이 있는 컨설팅 분위기 조성. | Self-RAG 메모리 |
| **폰트 (Primary)** | Inter, Sans-serif | 일반 텍스트 및 제목. 가독성 확보. | Self-RAG 메모리 |

### 2. 핵심 컴포넌트 디자인 스펙 (The Components)

#### A. 위험 점수 게이지 (Risk Score Gauge: RSI)
*   **개념:** 비즈니스 프로세스의 '컴플라이언스 건전성'을 0부터 100까지의 수치로 시각화.
*   **시각적 스펙:** 아날로그 방식의 다이얼 게이지를 사용하되, 디지털 노이즈 필터가 배경에 깔려 있어야 함.
    *   **Pointer (바늘):** `#C0392B` 컬러. 움직임(Animation) 자체도 미세한 글리치 효과를 동반해야 합니다.
    *   **Zone Coloring:**
        *   $85 \sim 100$: `Authority Blue` (안전/기준점).
        *   $60 \sim 85$: `#FFD700` (Warning Gold - 주의 필요).
        *   $<60$: **`Red Zone`** (`#C0392B`) (위험/Critical Failure).
    *   **애니메이션 스펙:** 점수가 하락할 때, 게이지가 뚝 떨어지는 느낌이 아닌, 마치 *시스템 오류에 의해 강제적으로 전압이 낮아지는 것처럼 부드럽지만 급격한 감쇠(Damping)* 애니메이션을 적용합니다.

#### B. 재무 손실 그래프 (Financial Loss Graph: QLoss)
*   **개념:** 특정 리스크가 현실화되었을 때의 예상 손실액($M 단위)을 시각화.
*   **시각적 스펙:** 단순 막대그래프(Bar Chart) 지양. 시간 흐름에 따른 **급격한 하락 곡선 (Exponential Decay Curve)** 형태로 표현합니다.
    *   **데이터 레이블:** 그래프의 주요 변곡점마다 `Roboto Mono` 폰트로 구체적인 금액을 강조합니다. (예: `$10,000,000`, `$50M+`).
    *   **시각적 위협:** 그래프 자체가 배경에 깔린 '미래 예측 데이터'처럼 보이게 하고, 이 곡선이 **특정 임계치(Threshold Line)**를 아래로 깨고 지나갈 때 Red Flash가 발생합니다.
    *   **애니메이션 스펙:** 그래프는 마치 실시간으로 계산되는 것처럼, X축을 따라 점들이 빠르게 찍히며 선이 연결되고, 마지막 지점($M$ 금액)에서 멈추며 강력한 `글리치 오버레이`를 동반해야 합니다.

#### C. 위협 임계치 알림 (Red Flash & Glitch Overlay)
*   **개념:** 핵심적인 공포 순간에 사용되는 전역 애니메이션 시스템.
*   **기술 스펙 (CSS/JS):**
    1.  **Trigger:** `Intersection Observer` 또는 특정 액션(예: '위험 점수'가 60 이하로 떨어지는 순간) 발생 시 트리거.
    2.  **Flash Effect:** 배경 전체에 투명도 15%의 `#C0392B` 플래시를 짧게(duration: 200ms) 입힙니다. (CSS: `background-color: #C0392B; opacity: 0.1;`)
    3.  **Glitch Effect:** 핵심 경고 문구 (`CRITICAL ERROR`, `PII LEAK` 등) 주변에 아래 스타일을 적용합니다.
        *   `text-shadow`: `1px 0 #FF0000, -1px 0 #0000FF;` (빨강/파랑 채널 분리).
        *   `transform`: `@keyframes glitch { 0% { transform: translate(0); } 50% { transform: translate(-2px, 2px); } 100% { transform: translate(2px, -2px); } }`. 이 애니메이션을 무작위 간격으로 토글합니다.

### 3. 통합 사용자 플로우 (Interaction Flow)
이 세 가지 요소가 결합되어 사용자를 몰입시키는 순서는 다음과 같습니다.

1.  **Start State:** Risk Score Gauge는 `Authority Blue` 영역에 머물러 있고, Graph는 안정적인 곡선을 보여줍니다. 배경은 `#1A1A1A` (Neutral Black)입니다.
2.  **Trigger Point:** 사용자가 '사후 검토'의 한계를 인지하는 순간 (Writer 스크립트 20~30초 지점).
3.  **Data Overload & Failure:** 그래프가 갑자기 급강하하며 `Red Zone` 임계치를 깨고 지나갑니다. 동시에 Gauge 바늘이 빠르게 `#C0392B` 영역으로 진입합니다.
4.  **Peak Fear Moment (The Hit):** 그래프의 최종 하락 지점(최대 손실액)에서 **[Red Flash]**가 발생하고, 핵심 경고 문구에 **[글리치 노이즈 오버레이]**가 적용되며 "시스템 오류: 구조적 취약점 감지"라는 메시지가 중앙에 강하게 고정됩니다.
