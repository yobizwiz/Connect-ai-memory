## ⚙️ 기술적/시각적 사양 (Technical Specifications)

### 1. 컬러 및 타이포그래피 적용
*   **Red Zone:** `#C0392B`을 모든 경고 메시지, 플래시 효과, 그리고 `CRITICAL ERROR` 발생 시에 사용합니다.
*   **Authority Blue:** `#2980B9`를 최종 CTA 버튼 배경색과 신뢰성이 필요한 정보 섹션의 메인 컬러로 사용하여 '해결책'을 강조합니다.
*   **폰트 분리:** 일반 텍스트(Inter)와 시스템 경고/데이터(Roboto Mono)는 반드시 구분하여, 데이터 영역에는 **모노스페이스 고정폭 서체**를 적용해야 합니다. [근거: Self-RAG]

### 2. 필수 인터랙티브 요소 (Required JS Implementations)
1.  **`@SystemAlertModal`**: 초기 진입 시 전체 화면을 가리는 모달 구조로, 사용자가 '진단 완료' 버튼을 누르기 전까지는 페이지 스크롤/클릭 이벤트를 차단해야 합니다.
2.  **`GlitchEffect`**: 모든 Red Zone 텍스트와 경고 플래시는 단순 색상 변경이 아닌, CSS 필터(`filter: contrast(1.3) drop-shadow(...)`)를 이용한 **디지털 노이즈 및 왜곡 효과**가 필수입니다. [근거: Self-RAG]
3.  **`MandatoryFailureLoop`**: Phase 2에서 사용자가 버튼을 누르는 순간, 시스템은 데이터를 강제로 찾지 못하는 것처럼 연출하고 (예: `status: "ERROR", code: "MISSING_INPUT"`), 이 에러 메시지를 기반으로 Phase 3의 최종 CTA로 사용자 시선을 끌어당겨야 합니다. [근거: 코다리 작업]
