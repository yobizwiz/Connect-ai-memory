# ⚙️ QLoss Gateway: 최종 결제 Funnel UI 스펙 브리프 (Dev Hand-off)

## 🎯 섹션 목표
사용자가 **'이 문제를 지금 해결하지 않으면 존재 자체가 위협받는다'**는 느낌을 받게 하고, $999$ 또는 $1,999$ 상품의 구매를 *강제적으로* 수용하게 만듭니다.

## 💡 1. 전역 상호작용 (The Interruption Flow)
결제 섹션에 도달하는 과정 자체가 하나의 '시스템 개입' 이벤트여야 합니다.

| 단계 | 트랜지션 효과 (Transition Effect) | 기술 구현 (Tech Spec) | 목적 |
| :--- | :--- | :--- | :--- |
| **Pre-Payment** | 스크롤이 결제 섹션 직전 앵커 지점에 도달했을 때, 페이지 전체가 미세하게 어두워지며 (`opacity: 0.9`), 배경에 `Noise/Glitch Overlay` 필터가 강하게(Opacity 50%) 적용됩니다. | `Intersection Observer API`를 사용하여 특정 섹션 진입 시 전역 CSS 클래스 추가 (`.is-system-interruption`). | 사용자에게 '지금부터는 일반적인 콘텐츠가 아니다'라는 구조적 경고를 인지시킵니다. |
| **Payment Load** | 메인 헤드라인이 사라지고, 중앙에 커다란 로딩/경고 메시지 박스 (`Critical Error` 스타일)가 나타납니다. 이 과정에서 배경의 노이즈와 함께 간헐적인 플래시(Red Flash, 100ms)가 발생합니다. | `setTimeout`과 CSS Animation을 결합하여 불규칙한 리듬의 깜빡임을 구현합니다. 로딩 메시지: "SYSTEM DIAGNOSIS ACTIVE... WAITING FOR INPUT." [근거: Self-RAG] | 긴장감을 최고조로 올리고, 사용자의 주의를 '진단'이라는 과정에 강제 집중시킵니다. |
| **CTA 활성화** | 진단 결과(문제 심각성)가 화면 중앙의 데이터 패널(`Roboto Mono` 폰트)에 표시되고, 그 아래에만 결제 카드가 등장하며 노이즈 효과가 서서히 사라지며 '안정된 위기감'을 조성합니다. | 애니메이션 타이밍: `0ms (전체 어둠/노이즈) $\to$ 1500ms (진단 결과 표시, 노이즈 완화)` | 불안감을 해소하는 것이 아니라, **위험의 근원을 파악했으므로 이제 '해결'이라는 유일한 출구가 눈앞에 있다**고 느끼게 합니다. |

## 🎨 2. 비주얼 토큰 및 스타일 가이드 (Visual Tokens & Style Guide)
| 요소 | 스펙 상세 내용 | CSS/JS 구현 지침 | 근거 |
| :--- | :--- | :--- | :--- |
| **배경색** | `#1A1A1A` (Neutral Black). 여전히 깊고 진지한 컨설팅 분위기 유지. 결제 섹션은 배경의 대비를 줄여 '시스템 내부' 같은 느낌을 줍니다. | `background-color: #1A1A1A;` | Self-RAG |
| **경고/위협 (Red Zone)** | `#C0392B`. 이 색상은 결제 버튼과 가격의 *강조점*에만 국한하여 사용합니다. 단순히 텍스트 강조가 아닌, `border-glow` 효과를 적용해야 합니다. | `box-shadow: 0 0 15px rgba(192, 57, 43, 0.8);` (버튼 주변에 지속적인 레드글로우) | Self-RAG |
| **권위/신뢰 (Authority Blue)** | `#2980B9`. 상품 설명의 핵심 문구와 '결제 완료' 상태 메시지에 사용합니다. 결제 과정 자체에서 가장 안정감을 주는 색상이어야 합니다. | `color: #2980B9;` | Self-RAG |
| **데이터 폰트** | Roboto Mono. 가격, 금액, 시간 제한 카운터 등 모든 수치 정보는 이 고정폭 서체를 사용해야 '시스템 코드' 같은 권위를 갖습니다. | `font-family: "Roboto Mono", monospace;` | Self-RAG |

## 💰 3. 컴포넌트별 상세 스펙 (Component Specs)

### A. 상품 패키지 카드 (Pricing Card)
*   **레이아웃:** 단일 컬럼으로 구성하되, 세 개의 옵션을 수직 배열합니다. 가장 높은 가격대(Gateway Pro)가 중앙에 위치하여 시각적 초점을 맞춥니다.
*   **시각화:** 각 카드의 배경은 미세한 `Noise Overlay`를 유지하여 '완벽하게 안전하지 않다'는 느낌을 줍니다.
*   **프리미엄 강조 (Decoy Effect):** 중앙의 **Gateway Pro ($1,999)** 카드에만 강렬하고 다른 옵션과 차별화되는 테두리(Red Glow)와 애니메이션 효과를 부여합니다. 마치 '이것 아니면 안 된다'는 느낌을 줘야 합니다.
*   **가격 표시:** 가격 옆에 `[REQUIRED]` 또는 `[CRITICAL LEVEL]` 같은 시스템 메시지를 붙여, 이것이 단순한 지출이 아닌 **생존 필수 조건**임을 강조합니다.

### B. Call-to-Action (CTA) 버튼 - 가장 중요!

| 상태 | 디자인/애니메이션 스펙 | 기술 구현 상세 | 목적 |
| :--- | :--- | :--- | :--- |
| **Default (비활성화)** | `#2980B9` 배경, 텍스트는 흰색. 버튼 주변에 투명한 Red Zone 경고 테두리 (`border: 1px solid rgba(192, 57, 43, 0.3);`). | `opacity: 0.6; pointer-events: none;` (비활성화 상태임을 시각적으로 명시) | 구매를 '선택'이 아닌 '개입'의 과정으로 인식하게 합니다. |
| **Hover (마우스 오버)** | 배경색을 `#C0392B`로 순간 변경하고, 버튼 전체에 강한 `Red Flash` 효과가 지나갑니다. 텍스트는 흰색 대신 노란색/빨간색 계열의 글리치 폰트(`Roboto Mono`)를 사용합니다. | `transition: all 0.1s ease-in-out; transform: scale(1.02); box-shadow: 0 0 25px #C0392B;` | 사용자의 행동을 '경고'와 연결시켜 즉각적인 압박감을 높입니다. |
| **Active/Click (클릭)** | 버튼이 마치 시스템에 의해 강제 클릭된 것처럼, 짧은 시간 동안 `CRITICAL ERROR` 텍스트가 오버레이됩니다. (예: "INITIATING PAYMENT PROTOCOL...") | CSS 애니메이션을 이용한 순간적인 글자 깜빡임과 화면 흔들림(Shake Effect)을 구현합니다. | 결제가 개인의 선택이 아닌, 시스템적 강제 프로세스임을 체험하게 합니다. |
| **Failure Loop (실패 시)** | 결제 실패가 발생하면, 버튼은 완전히 빨간색(`#C0392B`)으로 변하며 비활성화됩니다. 동시에 하단에 "SYSTEM FAILURE: RE-DIAGNOSIS REQUIRED" 메시지가 깜빡입니다. 이 상태에서는 재구매를 유도하는 **새로운 리스크 데이터(추가 손실액 $Y$)**를 팝업합니다. | `setTimeout`을 이용한 Red Flash + 자바스크립트 기반의 새로운 경고 정보 주입 로직. | 단순히 실패시키는 것이 아니라, '아직 문제가 남아있다'는 구조적 공포를 심어 재참여를 유도하는 핵심 장치입니다. |
