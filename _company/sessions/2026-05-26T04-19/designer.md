# 📊 Loss Meter MVP: 최종 인터랙티브 UX/UI 브리프 (v2.0)
**(Goal: 사용자가 '안전하지 않다'는 구조적 공포를 느끼고, 유료 진단 체험(CTA)을 강제 수용하게 만든다.)**

## I. 시스템 개요 및 목표 원칙
*   **목표:** 사용자의 현재 상태를 '구조적으로 취약함'으로 정의하고, 이 위험도를 **재정적 손실($X M)**로 즉각 치환하여 공포감 극대화.
*   **톤앤매너:** 권위적(Authoritative), 긴급함(Urgent), 위압적(Imposing). (Self-RAG 근거: Deep Slate Blue와 Red Zone의 조합)
*   **기술 원칙:** 모든 주요 전환점에는 **강제적인 인터랙티브 지연 시간(Forced Delay)**을 삽입하여 사용자가 '데이터 처리 중'이라는 혼란과 불안정성을 느끼게 해야 합니다.

## II. Loss Meter 사용자 여정 (User Flow & State Transitions)

| # | 단계 (State) | 목적 | 핵심 액션/UI 요소 | 개발 지시사항 (Technical Requirement) |
| :---: | :--- | :--- | :--- | :--- |
| **1** | **[LOAD] 초기 진입 및 위협 조성** | 시스템의 무작위성과 불안정성을 체감하게 한다. | 🔹 전체 화면에 `Noise/Glitch Overlay` 필터 적용 (투명도 10%).<br>🔹 로딩 바는 단순 막대 X, 파동 형태의 **불안정한 애니메이션** 사용.<br>🔹 배경에 고전적인 컴플라이언스 플로우차트(Before)를 흐릿하게 배치. | `CSS Filter` + `setInterval` 기반으로 낮은 빈도의 노이즈와 색상 왜곡을 지속적으로 발생시켜야 함. 로딩 바는 CSS Keyframe 애니메이션을 사용하되, 예측 불가능한 파동 패턴을 가져야 함. |
| **2** | **[ANALYSIS] 데이터 분석 및 경고 발생** | 문제의 복잡성과 심각성을 데이터 기반으로 충격적으로 제시한다. | 🔹 `Loss Meter` 게이지가 **0%에서 빠르게 상승(Acceleration)**하는 애니메이션.<br>🔹 임계점 도달 시, 시스템이 강제로 빨간 플래시를 일으키며 경고 메시지(`⚠️ CRITICAL FAILURE`) 출력. (Self-RAG 근거)<br>🔹 **데이터 로드/경고 문구는 반드시 `Roboto Mono` 고정폭 서체 사용.** | 게이지 애니메이션은 단순 선형 상승이 아닌, 초기에는 느리게 시작했다가 속도가 급격히 빨라지는 가속도 곡선(Exponential Curve)을 가져야 함. 플래시 효과는 200ms의 강렬한 `opacity: 1` 전환 후 즉시 `opacity: 0`으로 돌아와야 함. |
| **3** | **[STALL] 구조적 리스크 체감 단계 (핵심)** | 사용자의 주의를 분산시키고, '현재 시스템에 문제가 있다'는 의심을 심어준다. **(가장 중요)** | 🔹 게이지 수치 도달 후, 화면이 일시적으로 정지하고 `Processing Data...` 문구가 나타남.<br>🔹 **[3초 강제 지연]:** 이 시간 동안 모든 인터페이스 요소에 미세한 떨림(Jitter/Shake) 애니메이션을 적용하여 시스템 불안정성을 강조. <br>🔹 배경의 Noise 필터가 최고조로 올라감 (Opacity 20%). | JavaScript `setTimeout()` 함수를 사용하여 **최소 3초간 강제적인 비활성화(Stall)** 구간을 의무화해야 함. 이 시간 동안 사용자에게 아무런 편안함이나 예측 가능성을 주어서는 안됨. |
| **4** | **[RESOLUTION] 해결책 제시 및 CTA 유도** | 공포를 해소할 수 있는 유일한 경로가 '유료 진단'임을 각인시킨다. | 🔹 배경의 Noise 필터와 Glitch Overlay가 Authority Blue (`#2980B9`) 계열로 전환되며 점진적으로 사라짐.<br>🔹 핵심 문구: "이 구조적 위험은, 시스템 외부에서만 측정 가능합니다." (강조)<br>🔹 CTA 버튼(`[LOSS METER 진단 신청]`) 주변에 미세한 **'권위적인 빛' 효과**를 추가하여 클릭을 유도. | 애니메이션 전환 시, Red Zone $\rightarrow$ Authority Blue로의 컬러 팔레트 변화가 명확해야 함. CTA 버튼은 단순 버튼이 아닌, 마치 중요한 시스템 명령어를 실행하는 듯한 굵고 견고한 디자인이어야 함. |

## III. 비주얼 및 기술적 세부 가이드 (Developer Checklist)

| 요소 | 상세 규격 | 사용 목적 및 효과 | 구현 참고 사항 |
| :---: | :--- | :--- | :--- |
| **메인 색상** | 🔴 Red Zone: `#C0392B` / 🔵 Authority Blue: `#2980B9` / ⚫ Neutral Black: `#1A1A1A` | 위협(Red) $\rightarrow$ 신뢰/해결책(Blue)의 극적인 대비를 통해 감정적 동선 유도. | 모든 배경색과 경고문구에 일관되게 적용되어야 함. |
| **폰트 시스템** | Primary: Inter (Sans-serif), Data/Alert: Roboto Mono (Monospace) | 일반 텍스트는 가독성 있는 전문성을, 경고 메시지는 '시스템 코드'의 권위를 부여함. | 모든 에러 메시지 및 수치 데이터에 `Roboto Mono` 적용 필수. |
| **글리치 효과** | Glitch Animation (`text-shadow: 1px 0 red, -1px 0 blue;`) | 단순한 애니메이션이 아닌, **디지털 시스템 오류가 발생하는 듯한 시각적 왜곡**을 통해 신뢰도를 무너뜨림. | H1 제목 및 모든 경고 메시지(`CRITICAL ERROR`, `WARNING`)에 적용해야 함. |
| **Loss Meter 수치** | $50M 규모의 구체적인 숫자 (`$X,XXX M`). | 추상적인 위험이 아닌, 사용자에게 즉각적으로 와닿는 '재정적 피해'로 변환하여 공포를 극대화함. | 숫자는 굵기(Weight)와 색상을 이용해 가장 강하게 강조해야 함. (e.g., **$50M**) |
