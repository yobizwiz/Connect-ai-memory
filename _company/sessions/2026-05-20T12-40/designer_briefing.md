# Designer 에이전트 요청: Loss Meter의 위협 강화 (v2.0 Update)

**목표:** 랜딩 페이지에 적용된 'Loss Meter' 컴포넌트를 단순한 재무 손실 시각화를 넘어, 세 가지 핵심 구조적 불안감(Structural Anxiety)을 대표하는 인터랙티브 공포 요소로 진화시켜야 합니다.

**핵심 원칙:**
1. **공포의 다변화:** Red Zone이 단순히 '돈'의 문제가 아닌, '통제력', '법적 권위', '데이터 무결성'에 관한 문제임을 시각적으로 증명해야 합니다.
2. **상호작용 강화:** 사용자가 각 위협 요소를 클릭하거나 상호작용할 때마다 경고음/글리치 노이즈가 발생하며, 리스크 점수($R_{score}$)와 연동되어야 합니다.

**세부 요구사항 (3가지 Red Zone 모듈):**
1. **🚨 출처 불명(Source Ambiguity) 존:** 데이터 근거의 '흐릿함'을 시각화합니다. (예: 노이즈가 가득한 텍스트 블록, 깨지는 글리치 효과). -> [위협 키워드] 투명성 결여
2. **⚠️ 프로세스 단절(Compliance Drift) 존:** 시스템의 '균열'과 '누락'을 시각화합니다. (예: 매뉴얼 페이지가 찢어지거나, 체크박스가 비활성화되는 애니메이션). -> [위협 키워드] 통제력 상실
3. **🔒 PII 유출(Leakage) 존:** 데이터의 '경계 파괴'를 시각화합니다. (예: 마스킹되어야 할 정보가 갑자기 선명하게 드러나는 순간, 경고 시스템의 오버플로우). -> [위협 키워드] 프라이버시 침해

**제작 결과물:** 위의 3가지 요소를 통합한 최종 UI/UX 에셋 패키지 및 해당 애니메이션 로직을 구현할 개발 가이드를 요청합니다.
