### 💡 LLM 시스템 현황 보고서 (FAQ)
**(Powered by Gemini, Secured by Risk Engine)**

#### ❓ Q1. 어떤 모델을 사용하나요?
**A.** 핵심 지능 엔진(Intelligence Core)으로는 Google의 **Gemini 계열 LLM**을 사용하고 있습니다. [근거: 세션 활동 기록]
*   **역할:** 방대한 양의 데이터를 분석하고, 다양한 형태의 콘텐츠 (스크립트, 카피라이팅, 기획안 등)를 초안으로 생성하는 역할을 합니다. [근거: 2026-05-21T14-49/secretary.md]
*   **핵심 기능:** 단순히 정보를 나열하는 것이 아니라, 복잡한 패턴을 찾아내고 새로운 아이디어를 구조화합니다.

#### ❓ Q2. 이 시스템의 가장 큰 특징은 무엇인가요? (가장 중요한 부분!)
**A.** 바로 **'리스크 엔진(Risk Engine)'**이라는 별도의 검증 레이어가 존재한다는 점입니다. [근거: 세션 활동 기록]
*   **작동 원리:** Gemini가 아무리 좋은 결과물을 내놓더라도, 이 결과물은 **반드시 리스크 엔진을 거쳐 구조적 무결성 검증(Structural Integrity Check)**을 받습니다. [근거: 2026-05-21T14-49/secretary.md]
*   **목표:** 생성된 모든 콘텐츠나 데이터가 우리 회사의 핵심 목표(재무 손실액($Y$) 공포 유발 → 진단 요청)와 일관되고 논리적 결함이 없는지 검증하는 것이 최우선입니다.

#### ❓ Q3. 개발 관점에서 어떤 방식으로 호출되나요?
**A.** 내부적으로는 파라미터 설정과 API 흐름을 정교하게 제어합니다. [근거: 세션 활동 기록]
*   **API 통제:** API 호출량이 급증하거나 시스템 과부하가 걸리지 않도록 에이전트별로 작업 순서를 명확히 분배하고, 호출 빈도를 조절하는 모니터링 대시보드가 구축되어 있습니다. [근거: 2026-05-21T14-49/secretary.md]
*   **결과:** 모델의 파워를 최대한 활용하면서도 안정적이고 예측 가능한 결과물을 얻는 것이 목표입니다.
