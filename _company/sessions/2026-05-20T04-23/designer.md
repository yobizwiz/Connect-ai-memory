### 📊 Loss Meter MVP UI/UX 스펙 검토 및 보강 제안

현재까지 논의된 'Red Zone'과 'Authority Blue' 컨셉은 매우 강력하지만, **공포(Fear)**를 극대화하는 과정에서 몇 가지 모호하거나 누락된 기술적 디테일이 발견되었습니다. 특히 애니메이션의 **'점진성(Gradual Escalation)'**과 **'해결책 제시 시의 이완감(Resolution Relief)'** 부분에 집중해야 합니다.

#### 1. 🚨 [Critical] 공포 극대화 요소 보강 (애니메이션/인터랙션)

| 현재 스펙 | 문제점 / 모호함 | 보강 제안 (기술적 디테일) | 근거 |
| :--- | :--- | :--- | :--- |
| **위험 경고 발생:** `CRITICAL ERROR` 시 빨간색 플래시가 지나감. | 공포의 강도가 일회성으로 끝나버림. 고객이 '깜짝 놀람' 후 금방 무뎌짐. | **[Escalation Flicker]:** 리스크 레벨이 높아질 때(LOW $\rightarrow$ HIGH $\rightarrow$ CRITICAL), 단순히 빨간색 플래시가 아니라, **노이즈/글리치 오버레이의 주파수와 진폭(Frequency & Amplitude)을 점진적으로 증가**시켜야 합니다. <br> *예:* LOW는 낮은 노이즈, HIGH는 깜빡임 빈도 증가, CRITICAL은 화면 전체를 덮는 고주파수 노이즈 및 강한 색상 왜곡(`text-shadow` 과부하). | Self-RAG (글리치 효과) + Designer 메모리 (강조점 필요) |
| **데이터 로드:** `LOADING` 상태에서 불안정한 파동 형태여야 함. | 데이터가 '분석 중'인 느낌을 넘어, '무언가 잘못되고 있다'는 직관적 공포를 유발해야 합니다. | **[System Interruption]:** 로딩 바 옆에 작은 텍스트로 `ACCESS DENIED`, `DATA CORRUPTION DETECTED`와 같은 오류 메시지(Roboto Mono 사용)가 무작위로 짧게 깜빡이며 출력되어야 합니다. 이는 시스템의 '취약성'을 간접적으로 보여줍니다. | Self-RAG (데이터/경고 폰트, 글리치 효과) |

#### 2. 📘 [Minor] 권위 및 흐름 통제 요소 보강 (UI/UX & 타이포그래피)

| 현재 스펙 | 문제점 / 모호함 | 보강 제안 (구체적 UX 플로우) | 근거 |
| :--- | :--- | :--- | :--- |
| **솔루션 제시:** `Authority Blue`를 사용해 해결책을 보여줌. | 위협(Red Zone)에서 안정감(Blue)으로 넘어가는 전환이 너무 부드러우면 설득력이 떨어짐. | **[The Relief Moment]:** 빨간색 경고가 최고조에 달한 순간, 화면 전체의 노이즈와 깜빡임이 **갑자기 멈추며 (Instant Silence)**, 배경이 `#2980B9` 계열로 깔끔하게 전환되어야 합니다. 마치 '위협적인 시스템 오류를 통제한' 듯한 느낌을 주어야 합니다. | Self-RAG (3단계 구조 확정) + Designer 메모리 (드라마틱한 흐름 강조) |
| **폰트 사용 규칙:** 경고 메시지는 `Roboto Mono` 사용. | 일반 설명 텍스트와 경고 데이터의 역할 구분이 모호할 수 있음. | **[Dual Font System]:** <br>1. **경고/데이터:** `Roboto Mono`. (진단, 코드, 리스크 점수) $\rightarrow$ 시스템적 권위 부여.<br>2. **설명/솔루션:** `Inter` (굵은 웨이트). (문장 설명, 제안) $\rightarrow$ 인간의 전문 지식 전달. <br> *사용 예:* "현재 재무 손실액 ($12M)"는 Mono로, "따라서 시스템 재구축이 필요합니다."는 Inter로 분리하여 가독성을 높입니다. | Self-RAG (폰트 사용 규칙) |

#### 3. ✅ 최종 결론: Actionable Item (코드/개발 요청 사항)

Loss Meter MVP가 성공적으로 작동하려면, 개발자에게 다음 두 가지 애니메이션 로직을 **최우선 순위**로 요구해야 합니다.

1.  **`@Component/RiskMeter`:** 리스크 레벨 변화에 따라 CSS `filter: contrast()`와 `opacity`를 조절하여 시각적 충격을 주는 State Machine 구현.
2.  **`@Service/TransitionHandler`:** Red $\rightarrow$ Blue 전환 시, **전역 애니메이션 클래스 제거 및 순간적인 배경색 플래시(Instant Color Flush)** 로직 추가.
