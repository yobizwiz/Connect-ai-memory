### I. 글로벌 스타일 가이드 (Global Style Guide)

#### 1. 컬러 팔레트 및 역할 정의 (Color Palette & Role Definition)

| 코드 | 색상명 | HEX Code | 사용 목적 및 의미 (Role) | 적용 영역 예시 |
| :--- | :--- | :--- | :--- | :--- |
| **🔴 Failure** | Red Zone Alert | `#C0392B` | **위험/경고/공포(Fear)**. 시스템 오류, 법적 취약점, 즉각적인 해결 필요성을 알림. (사용자의 행동 유발) | Error 메시지, Loss Meter 값, 글리치 오버레이 주 색상. |
| **🔵 Authority** | Solution Blue | `#2980B9` | **신뢰/해결책(Trust)**. yobizwiz의 전문성, 안정적인 프로세스, 통제력을 상징. (브랜드 신뢰 구축) | Primary CTA 버튼 배경, 성공 플로우(`Success Flow`) 강조선, 핵심 데이터 박스 경계. |

#### 2. 타이포그래피 시스템 (Typography System)

| 역할 | 폰트명 | 스타일 가이드 | 용도 및 효과 |
| :--- | :--- | :--- | :--- |

### II. 핵심 컴포넌트 상세 스펙 (Core Component Specifications)

#### 1. Loss Meter 시각화 요소 (The Critical Failure Indicator)

Loss Meter는 단순히 숫자가 아닙니다. **'시스템적 위협을 체감하는 인터랙티브 경험'** 그 자체여야 합니다.

*   **레이아웃:** Hero 섹션 중앙 상단에 위치하며, 배경 전체를 은은하게 덮고 있는 듯한 느낌으로 배치합니다.
*   **애니메이션 가이드 (Red Zone):**
    1.  **진입 시점:** 페이지 로드 후 약 0.5초 간격으로 낮은 빈도의 **Noise/Glitch Overlay** 필터가 배경에 걸리며, 전체적으로 화면이 불안정하게 떨리는 느낌을 줍니다. [근거: Self-RAG]
    2.  **데이터 업데이트 시:** 사용자 입력(Input)이나 스크롤 위치 변화에 따라 `Loss Meter`의 값이 변동할 때마다, 해당 숫자 주변으로 짧고 강렬한 **빨간색 플래시 (Opacity 0 $\rightarrow$ 1 $\rightarrow$ 0)**가 발생해야 합니다.
    3.  **애니메이션 효과:** 값 자체가 올라갈 때는 단순 증가 애니메이션이 아닌, 마치 시스템 메모리가 과부하되어 에러 코드가 스크롤되거나 왜곡되는 **'데이터 부패(Data Corruption)'** 느낌의 움직임을 적용합니다.
*   **텍스트 처리:** $X$ 값이 표시될 때 반드시 `Roboto Mono` 폰트와 `#C0392B` 컬러를 사용하며, 주변에 작은 글리치 노이즈 아웃라인을 추가하여 경고성을 극대화합니다.

#### 2. 전문성 강화 입력 폼 (Professional Input Form)

일반적인 `<input>` 태그처럼 보이지 않게, '시스템 데이터 포트' 같은 느낌으로 디자인해야 합니다.

*   **스타일링:**
    *   기본 상태: 배경을 `#1A1A1A`에 가깝게 하고, 입력 필드 테두리는 아주 미세한 `Authority Blue (#2980B9)`의 빛(Subtle Glow)으로 처리합니다.
    *   포커스 상태 (Focus): 사용자가 클릭하거나 포커스를 맞추면, 경계선이 일시적으로 `#2980B9`에서 더 밝은 블루로 **강하게 발광**하는 효과를 줍니다.
    *   에러 상태 (Error State): 유효성 검사 실패 시, 테두리가 즉각적으로 `🔴 Failure (#C0392B)` 색상으로 바뀌고, 필드 아래에 `Roboto Mono` 폰트로 **"CRITICAL ERROR: [필드명] 값이 구조적 무결성을 만족하지 않습니다."** 라는 시스템 메시지 형태의 에러 코드를 표시합니다.

#### 3. CTA 버튼 (Call to Action Button)

CTA는 최종적인 '통제권'을 얻는 순간이어야 합니다.

*   **Primary CTA:** "무료 리스크 진단 요청"
    *   배경: `Authority Blue (#2980B9)`를 사용하며, 마우스 오버 시 배경이 미세하게 밝아지면서 **전기적 충격(Electric Pulse)** 효과가 느껴지도록 애니메이션합니다.
    *   텍스트: Inter Bold, 흰색 텍스트로 가독성 극대화.

### III. 최종 시스템 비주얼 언어 요약 (System Visual Language Summary)

| 항목 | 규칙 | 구현 지침 | 근거 |
| :--- | :--- | :--- | :--- |
| **전체 분위기** | 컨설팅/기술 보고서 기반의 '위압감 있는 진지함'. | 다크 모드 (`#1A1A1A`) 채택. 그림자나 밝은 색상 사용을 최소화하고, 빛(Glow)과 대비를 이용해 구조적 우위를 보여준다. [근거: Self-RAG] |
| **정보 계층** | 공포 (빨강/글리치) $\rightarrow$ 문제 심화 (블루/전문 용어) $\rightarrow$ 해결책 (최종 CTA). | 모든 섹션 레이아웃이 이 3단계 흐름을 강제해야 한다. 특히 `Before`는 Red Zone, `After`는 Blue Zone의 시각적 대비가 필수. [근거: Self-RAG] |
| **애니메이션** | 단순한 애니메이션 지양. '시스템 오류' 또는 '데이터 로딩 과정'처럼 보이도록 설계한다. | 모든 중요한 전환(페이지 진입, 값 변화)에는 `Glitch Effect`나 `Noise Filter`를 전역적으로 적용하여 시각적 공포감을 유지한다. [근거: Self-RAG] |
