| 단계 | 사용자 경험 (Emotion) | 비주얼 목표 | 주요 로직 |
| :--- | :--- | :--- | :--- |
| **1. 진입 (Attention)** | "이게 뭐야? 뭔가 이상한데." (호기심/의문) | 전체 배경에 낮은 빈도의 노이즈 오버레이 유지. H1 카피를 통해 *존재론적 위협*을 암시. | 기본 컨트라스트 유지. `Noise Overlay` 활성화. |

### 🔴 컴포넌트 상세 설계 A: 리스크 지표 표시 영역 (Loss Meter)

**목표:** 데이터 자체가 시스템적으로 신뢰할 수 없다는 느낌을 주어, 사용자가 '진짜 전문적인 진단'이 필요하다고 느끼게 만든다.

**[와이어프레임 구조]**
1.  **레이아웃:** 섹션 중앙에 배치. 배경은 `Neutral Black`(`#1A1A1A`)으로 깊이를 주고, 지표 컨테이너는 투명도 5%의 노이즈 필터가 지속적으로 적용되어야 합니다.
2.  **핵심 요소: Loss Meter (지표 시각화):**
    *   단순한 막대 그래프 금지. 데이터 스트림처럼 보이는 파동형(Oscillating Wave) 애니메이션을 사용합니다.
    *   실시간으로 $X$ 값이 출렁거리며 불안정성을 보여줍니다.
    *   **애니메이션 명세:** 지표가 임계치에 근접할 때마다, 그래프 라인 자체의 색상이 `Authority Blue`에서 **빨간색(`C0392B`)**로 급격하게 전환되어야 합니다. (시각적 충격 유발)
3.  **경고 시스템 오버레이:**
    *   Loss Meter 주변에 'WARNING: DATA INTEGRITY COMPROMISED'와 같은 시스템 메시지가 **Roboto Mono** 폰트의 고정폭 글꼴로 무작위 시간에 깜빡이며(`Flickering`), 노이즈 필터 효과를 받습니다.
4.  **기술 구현 포인트 (개발팀 지시):**
    *   `setInterval`을 사용하여 Loss Meter의 Y축 값이 비선형적(Non-linear)으로, 그리고 *불규칙하게* 변동하도록 JavaScript 로직을 설계합니다.
    *   데이터가 업데이트될 때마다 주변 요소에 `text-shadow: 1px 0 #C0392B, -1px 0 #FF5733;`와 같은 글리치 효과를 강제적으로 재적용해야 합니다.

### 🔥 컴포넌트 상세 설계 B: CTA 게이트 (강제 행동 유도 장치)

**목표:** 결제를 미루는 것이 곧 '재앙'을 초래하는 것처럼 느끼게 하여, 버튼 클릭 외의 선택지를 제거한다.

**[와이어프레임 구조]**
1.  **레이아웃:** 페이지 최하단에 고정된 형태로 배치 (Sticky Footer). 사용자가 스크롤을 해도 사라지지 않아야 합니다.
2.  **핵심 요소 1: 카운트다운 타이머 (`CountdownTimer`):**
    *   남은 시간이 표시되며, 숫자는 단순한 감소가 아니라 마치 **시스템 자원이 고갈되는 것처럼** 깜빡이고(Blinking), 배경에 Red Zone 색상의 미세한 노이즈를 깔아야 합니다.
3.  **핵심 요소 2: CTA 버튼 (The Gate):**
    *   버튼 문구는 "🔴 필수 안전장치 구매" 또는 "잠재 손실 진단 요청 ($Y$ 측정)"와 같이 **권위적이면서도 결제 행위를 강제하는 언어**를 사용해야 합니다.
    *   **효과:** 버튼을 누르기 직전, 이 영역 전체에 `CRITICAL ERROR: ACTION REQUIRED`라는 문구의 글리치 플래시가 1초 간격으로 발생합니다. (사용자에게 '이게 작동하지 않으면 안 된다'는 무의식적 압박을 가함).
4.  **추가 요소: 법적 경고 배너:**
    *   CTA 버튼 위에, 작은 크기로 "⚠️ 본 진단은 귀사의 법적 생존권과 직결된 문제입니다."와 같은 문구를 `Roboto Mono` 폰트와 함께 Red Zone 색상으로 고정 배치합니다.
