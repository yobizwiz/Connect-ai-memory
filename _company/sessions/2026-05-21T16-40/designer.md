# 🚨 DiagnosticEngine: Critical Failure Flow Design Brief (Mobile Optimized)

## 🎯 목적 및 목표
*   **목적:** 사용자가 시스템상의 '사소한 오류'로 인지하는 대신, **"내 비즈니스 프로세스 자체에 구조적이고 법적인 결함이 있다"**고 느끼게 만드는 경험을 제공합니다.
*   **핵심 감정 흐름:** 경악 (Shock) $\rightarrow$ 불안/혼란 (Confusion) $\rightarrow$ 안도감/절박함 (Relief/Urgency).
*   **주요 KPI:** 사용자가 '오류'를 해결하는 대신, **"전문가의 도움이 필요하다"고 느끼며 CTA(진단 요청) 버튼을 누르는 것**.

## 📱 I. 모바일 화면 흐름 (Screen Flow Map)
이 플로우는 하나의 긴 인터랙션으로 설계되어야 합니다. (총 예상 소요 시간: 10~20초)

### Step 0: 정상 상태 (Normal State)
*   **배경:** `#1A1A1A` (Neutral Black). 일반적인 서비스 화면처럼 보입니다.
*   **액션:** 사용자가 리스크 진단 기능을 실행하거나, 데이터를 업로드하는 등 '일반적'인 행위를 합니다.

### Step 1: 충격 유발 - 에러 발생 (The Trigger)
*   **트리거:** 백엔드 API 호출 실패 또는 구조적 데이터 불일치 감지 시.
*   **비주얼 효과:** 화면 전체가 찰나의 순간(200ms) 동안 강렬한 **Red Zone 플래시**를 일으킵니다. 이와 동시에 낮은 빈도의 노이즈 필터가 배경에 걸립니다.
*   **메인 요소:** 모달 경고창 (팝업 오버레이).
    *   **헤딩:** Writer 카피 중 가장 충격적인 문구 사용 예: `[LEGAL NULLIFICATION ALERT: CODE L-404]`
    *   **설명:** 시스템 오류 코드를 전문 용어로 풀어 설명합니다. "단순한 연결 실패가 아닙니다. 이는 **프로세스 무효화 가능성**을 의미합니다." (Red Zone 컬러 사용)
    *   **애니메이션 지시:** 경고 메시지는 텍스트가 한 글자씩 깨지듯(Glitch effect) 나타나야 합니다.

### Step 2: 혼란 조성 - 데이터 오버로드 (The Dread)
*   **트리거:** 사용자에게 '스스로 해결할 수 있는' 정보를 제시하는 단계입니다.
*   **비주얼 효과:** 모달 경고창이 사라지고, 배경 화면에 **시스템 로그/데이터 스트림**이 빠르게 스크롤됩니다 (Pseudo-hacker feel).
    *   로그 내용은 실제 데이터와는 무관하며, '결함'을 암시하는 전문 용어들로 가득 채웁니다. 예: `[ERROR_CODE 503]: Data Source Unreachable`, `COMPLIANCE_PROTOCOL_FAIL`. (Roboto Mono, Green/Red 색상으로 혼재)
    *   **오버레이:** 화면 상단에 작은 위젯 형태로 **'시스템 무결성 점수(System Integrity Score)'**가 표시되는데, 이 수치가 90점 $\rightarrow$ 35점으로 급락하는 시각적 애니메이션을 보여줍니다. (Red Zone 컬러 사용)
*   **핵심 메시지:** "이 오류는 사용자 환경의 문제가 아닙니다. **시스템 아키텍처 자체에 결함이 존재합니다.**"

### Step 3: 전환 및 해결책 제시 (The Pivot & CTA)
*   **트리거:** 데이터 오버로드 상태가 최고조에 달한 후, 강제로 인터페이스가 정리됩니다.
*   **비주얼 효과:** 배경 노이즈와 로그 스트림이 갑자기 정지하며(Freeze), 화면 중앙 하단에 **Authority Blue 색상의 명확하고 큰 버튼 영역**이 나타납니다. (시각적 안도감 유도)
*   **메인 요소:** CTA 패널.
    *   **헤딩:** "진정한 리스크를 해결할 아키텍처가 필요합니다." (권위적 톤)
    *   **버튼:** **[Yobizwiz Diagnostic Engine을 통한 구조적 무결성 진단 요청]** (Authority Blue 배경, 대비되는 흰색/검은색 텍스트).
