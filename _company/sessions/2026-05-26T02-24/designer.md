### 📑 [최종 산출물] QLoss Meter MVP: 결제 Funnel UI/UX Spec Brief v1.0
**(대상 페이지: 최종 결제 완료 전 단계)**

**목표:** 사용자가 돈을 지불하는 행위를 '개인의 선택'이 아닌, '시스템적 위협으로부터의 필수적인 방어 조치(Structural Intervention)'로 인식하게 만들어 구매 전환 강제성을 극대화한다.
**핵심 테마:** 구조적 취약성 $\rightarrow$ 시스템 경고 (Red Zone) $\rightarrow$ 권위적 해결책 수용.

#### ⚙️ A. API 연동 로직 및 데이터 흐름 정의
| 단계 | 트리거 | 입력 데이터 | 출력 결과물 | 디자인 적용 요소 |
| :--- | :--- | :--- | :--- | :--- |
| **1. 진단 완료** | 사용자가 '리스크 보고서 다운로드' 버튼 클릭 (혹은 자동 리디렉션) | `Report_ID`, `Loss_Amount` (X Million), `Structural_Flaw_Count` | Loss Meter API 호출 시작. 로딩 상태와 함께 임시 Red Zone 오버레이 적용. | **글리치/노이즈 오버레이**, 스피너 대신 '시스템 분석 중...' 텍스트 깜빡임. |
| **2. 리스크 계산** | API가 `Loss_Amount` 및 `Flaw_Count` 반환 (딜레이 3초 필수) | `{ "loss": X Million, "flaws": N }` | 계산된 손실액을 극대화하여 시각화하고, 이를 막기 위한 '필수 방어벽' 비용 제시. | **Red Zone 플래시**, `Roboto Mono`를 활용한 경고 수치 표시, 굵은 글자 변화. |
| **3. 구매 강제 유도** | 결론 도출 (솔루션 필요성 강조) | - | "이 리스크는 방관할 수 없습니다." 메시지 + CTA 버튼 활성화. | 배경의 권위적 블루와 경고색을 대조시키며 긴급성을 조성. |

#### 🎨 B. 섹션별 상세 UI/UX 스펙 (프레임 단위)

##### 1. [Pre-Payment Section] - '경고' 단계
**목표:** 사용자의 감정적 불안을 최고점으로 끌어올리고, 해결책 없이 이탈할 경우의 재무적 공포를 상기시킨다.
*   **배경/톤:** 전체 배경은 `Neutral Black` (`#1A1A1A`) 유지. 하지만 화면 중앙에만 **강렬한 노이즈 오버레이 필터 (Opacity 20%, Noise/Chromatic Aberration)** 적용.
*   **헤드라인 (H1):** `⚠️ 시스템 경고: 당신의 현 상태는 허용 가능한 범위가 아닙니다.` (`Red Zone` 컬러 사용)
    *   *(애니메이션)* H1이 로딩되자마자 텍스트에 짧은 간격으로 깜빡이는(flicker) 애니메이션을 적용한다.
*   **Loss Meter 시각화 (핵심):** API 결과가 도착하면, 페이지의 중앙 상단에 **'Estimated Loss Profile: [X Million]'** 박스를 강제로 고정 배치한다.
    *   *(애니메이션)* 이 금액은 일반 숫자가 아닌, 데이터 스트림처럼 위에서 아래로 떨어지며(Cascade), 각 자리마다 짧게 '글리치 노이즈'가 지나가는 효과를 준다. (`Roboto Mono` 폰트 필수).
*   **문제 진단 요약:** 발견된 `Structural Flaws (N개)` 리스트를 제시하되, 단순 목록이 아닌 **"잠재적 위협 영역"**이라는 제목으로 Red Zone 배경과 함께 배치한다.

##### 2. [Payment Form Section] - '시스템 개입' 단계
**목표:** 결제 과정 자체가 시스템의 강제적인 승인 절차처럼 느껴지게 만든다.
*   **섹션 타이틀:** `[SYSTEM OVERRIDE REQUIRED]` (권위적 Blue와 경고색을 섞어 사용).
*   **텍스트 메시지:** "이 구조적 리스크를 해소하려면, 즉각적인 시스템 개입(Service Fee)이 필요합니다. **진단된 손실액 $X Million$ 방어 비용은 최소 $1,999$입니다.**" (강제성 부여).
*   **결제 폼 배치:** 결제 카드 정보 입력 필드는 마치 '시스템 접속 포트'처럼 보이게 디자인한다. 배경에 미세한 `Circuit Board` 패턴을 오버레이하고, 입력창 주변에 파란색(Authority Blue)의 은은한 빛이 흐르는 애니메이션을 적용한다.
*   **구매 버튼 (`CTA`) 스펙 (가장 중요):**
    *   **텍스트:** "구조적 리스크 방어벽 활성화 및 진단권 구매 ($1,999$) - **지연 시 $X Million$ 추가 손실 예상**"
    *   **배경색:** 일반적인 CTA 색상이 아닌, `Authority Blue`와 `Red Zone`이 혼합된 '긴급 경고' 계열의 그라디언트를 사용한다.
    *   *(애니메이션)* 마우스 커서가 버튼 위에 올라가면, 버튼 주변으로 미세한 **글리치(Glitch)** 효과가 발생하며, 텍스트에 "⚠️ 지금 즉시 행동하세요"라는 경고 문구가 잠시 깜빡거려야 한다.

##### 3. [Confirmation/Processing] - '결정적 순간' 단계
**목표:** 결제 버튼 클릭 직후부터 성공까지의 모든 과정을 긴장감 넘치는 시스템 처리 과정으로 연출한다.
*   **클릭 액션:** 사용자가 구매 버튼을 누르는 즉시, 페이지 전체가 어두워지며 **강렬한 빨간색 플래시 (Opacity 0 $\rightarrow$ 1 $\rightarrow$ 0)** 가 짧게 지나간다.
    *   *(사운드 스펙)* 낮은 주파수의 '삐빅-' 하는 경고음(System Warning Tone)을 재생한다.
*   **로딩 상태:** 일반적인 로딩 바가 아니라, **'RISK MITIGATION IN PROGRESS...'** 라는 텍스트와 함께 불안정한 파동 형태의 노이즈 오버레이가 화면 전체를 감싼다.
    *   *(애니메이션)* 이 과정에서 `Loss Meter`의 X Million 수치가 다시 한번 '최대 리스크 구간(Red Zone)'을 가리키며 깜빡이는 시각적 연출을 삽입한다. (진단이 끝나지 않았다는 메시지를 전달).
*   **성공 화면:** 단순 "결제 완료"가 아닌, **`[SUCCESS: System Protocol Activated]`** 라는 문구와 함께, 불안정했던 배경 노이즈가 서서히 사라지고, 대신 안정적이고 깨끗한 `Authority Blue` 톤의 인터페이스가 잠시 나타난다.
*   **최종 CTA:** 성공 페이지 하단에 작은 글씨로 **"진정한 방어는 지속적인 시스템 모니터링을 필요로 합니다. (Upgrade to Annual Plan)"** 이라는 문구와 함께 연간 구독 유도 CTA를 배치한다.
