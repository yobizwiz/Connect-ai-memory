# 🚨 SYSTEM WARNING REPORT: 최종 디자인 스펙 v2.0 (Lead Designer)

## I. 프로젝트 개요 및 목표 [Self-RAG 근거]
*   **프로젝트 목적:** 고객의 현황 분석을 통해 단순한 '준수 미비'가 아닌, **'시스템적 무효화 위협(Structural Invalidity Threat)'** 상태임을 공포감 있게 인지시키고, yobizwiz만이 유일한 해결책(보험 상품)임을 각인시킨다.
*   **톤앤매너:** 하이테크(Hi-Tech), 경고(Warning), 권위적(Authoritative), 비극적 긴급성(Dramatic Urgency).
*   **핵심 메시지 플로우 (3막 구조):**
    1.  **Act 1: 공포 유발 (Red Zone)** - "당신은 위험합니다." (글리치, 경고음)
    2.  **Act 2: 문제 심화 (Data Overload)** - "문제는 당신이 생각하는 것보다 복잡하고 크다." (복잡한 데이터 시각화, QLoss 제시)
    3.  **Act 3: 해결책 제시 (Authority Blue)** - "우리가 유일하게 시스템을 재설계할 수 있습니다." (명료한 플로우차트, CTA)

## III. 핵심 컴포넌트 상세 디자인 스펙 (Interactive)

### 1. ⚠️ 위험 지수 게이지 (Risk Index Gauge Component)
*   **역할:** 보고서의 가장 첫 페이지 또는 섹션 전환 시 배치되어, 고객에게 '현재 상태가 안전하지 않다'는 즉각적인 심리적 충격을 준다. [근거: Self-RAG]
*   **구현 방식 (Interactive):**
    1.  **Initial State:** 게이지 바의 70% 이상이 Red Zone 색상(위협)으로 채워지고, 그 위로 `CRITICAL ERROR` 경고 메시지가 깜빡임. [근거: Self-RAG]
    2.  **Animation Trigger:** 보고서가 로딩되거나 고객 이름이 시스템에 입력되는 시점에, 게이지 바가 **불안정하게 파동치는 애니메이션**을 동반하며 위협 지수를 표시해야 함.
    3.  **Data Mapping:** '구조적 무결성 위협 TOP 5'의 종합 위험도를 계산하여 (예: 평균 $78\%$), 이 수치를 게이지에 실시간으로 오버레이 한다.

### 2. 🌊 데이터 플로우 차트 (Threat Flowchart)
*   **역할:** '문제 발생 전(Before)'과 '솔루션 적용 후(After)'의 극명한 대비를 통해 yobizwiz의 가치를 증명한다. [근거: Self-RAG]
*   **레이아웃 구성:** 3분할 구조 (A $\rightarrow$ B $\rightarrow$ C)
    *   **좌측 (Before - The Chaos):** 복잡한 노드와 화살표로 이루어진, 무정형의 '규제 준수 프로세스'. 배경에 미세한 글리치 효과를 넣고, 가장자리에 Red Zone 경고 테두리를 적용하여 **'무질서함'**을 시각화한다.
    *   **중앙 (The Threat):** `CRITICAL FAILURE` 메시지 박스를 강렬하게 중앙에 배치하고, 이 실패가 발생한 원인(예: API Scope 누락)을 Data/Alert Font로 노출한다. **이 섹션에서 페이지 전체의 빨간 플래시 효과를 200ms 동안 트리거 한다.** [근거: Self-RAG]
    *   **우측 (After - The Solution):** yobizwiz가 설계한 단순하고 명쾌하며 구조화된 '방어 아키텍처' 다이어그램. Blue Authority Color와 간결한 직사각형 블록을 사용하여 **'안정성'**과 **'해소감'**을 제공한다.

### 3. 📊 리스크 테마별 보고서 섹션 (TOP N Threats)
*   **구조:** 각 위협(예: GDPR 미준수, API Scope 오류)은 독립된 '경고 모듈'로 디자인되어야 한다.
*   **구성 요소:**
    1.  **헤딩:** `🚨 [위협 테마] - Level: CRITICAL` (Red Zone 컬러 및 글리치 효과 적용). [근거: Self-RAG]
    2.  **위협 설명:** 일반 텍스트(Inter)로 작성되나, '법적 근거'가 언급될 때는 `Roboto Mono`를 사용하여 시스템 로그처럼 보이게 한다.
    3.  **재무 손실액 제시 (The Hook):** 섹션 하단에 별도의 박스(`Total Risk Estimate`)를 만들고, 이 곳에 **구체적인 예상 재무적 손실액 ($XXX Million)**을 가장 큰 글자 크기(H1급)로 Red Zone 컬러와 함께 배치한다.

## IV. 디자인 구현 및 기술 사양 요약
*   **배경:** 전반적으로 `#1A1A1A` (Neutral Black)의 다크 모드 환경 유지.
*   **글리치/노이즈 효과:** 단순한 배경 패턴을 넘어, 특정 텍스트나 박스가 **'오류가 발생한 것처럼 왜곡되는 애니메이션 로직'**으로 구현되어야 한다. (CSS Filter + JavaScript Timing 이용) [근거: Self-RAG]
*   **CTA 버튼:** 단순 링크가 아닌, `[Execute Diagnosis]`와 같은 문구와 함께 API 호출을 연상시키는 **디지털 버튼 디자인**을 적용하며, Blue Authority Color를 사용한다.
