## ✨ II. 핵심 인터랙션 스펙: 글리치 & 지터링 모듈 [근거: 코다리 작업, Designer 메모리]

Red Zone 요소가 활성화될 때 반드시 작동해야 하는 3가지 계층적 효과입니다.

### 1. Glitch Overlay (시각적 노이즈)
*   **기능:** 모든 경고 메시지 영역에 전역적으로 투명도(Opacity) 20%의 `Noise/Glitch Overlay` 필터를 적용한다. [근거: Self-RAG]
*   **애니메이션 지침:** 단순한 색상 변화가 아닌, 디지털 시스템 오류 메시지 같은 **색상 왜곡 (Chromatic Aberration)**과 낮은 빈도의 노이즈(Noise)를 배경 전체에 겹친다.
*   **CSS 구현 가이드:** `filter: url(#glitch-effect)` 또는 CSS Filter와 Gradient를 활용하여 RGB 채널 분리 효과를 준다.

### 2. Jittering Effect (불안정성 떨림)
*   **Trigger:** QLoss 점수가 임계치(예: $50\% \rightarrow 75\%$ 구간)에 도달할 때마다 주기적으로 발생한다.
*   **지침:** 단순한 CSS `shake` 애니메이션을 넘어, **폰트의 가로/세로 위치가 불규칙하게 미세하게 떨리는 (Jittering)** 효과를 적용해야 한다. 이는 시스템이 정상 작동하지 않는다는 느낌을 주어야 함.
    *   **속도 곡선:** 초기 (50%): 느리고 주기적인 진동. 중기 (75%): 빠르고 불규칙한 픽셀 단위 떨림.
*   **CSS 구현 가이드:** `transform: translate(Xpx, Ypx)`를 무작위 값으로 짧은 시간 간격(`10ms ~ 50ms`) 동안 반복하여 적용한다.

### 3. Red Flash & Flicker (경고 발생)
*   **Trigger:** '법적/시스템 무효화 경고' 카피가 화면에 등장할 때 (클라이맥스). [근거: 코다리 작업]
*   **지침:**
    1.  **전역 플래시:** 페이지 전체 배경이 200ms 동안 강하고 빨간색(`rgba(192, 57, 43, 1)`)의 빛으로 빠르게 깜빡여야 한다. (Opacity $0 \rightarrow 1 \rightarrow 0$).
    2.  **깜빡이는 경고:** 핵심 문구 (`⚠️ CRITICAL ERROR`)는 일반 애니메이션 대신, 마치 시스템 메모리 오류처럼 짧은 간격(50ms)으로 강하게 깜빡여야 하며, 글리치 효과와 함께 노이즈가 최대화된다.

## 🌐 III. 매체별 적용 가이드라인 (Modularization)

| 매체 | 주요 특징 및 제약 | Red Zone 구현 방식 |
| :--- | :--- | :--- |
| **Web UI / Landing Page** | 인터랙티브, 실시간 피드백 필수. JavaScript 기반의 API 연동이 가장 중요함. | **실제 시스템 오류 시뮬레이션.** QLoss 데이터 변화에 따라 Jittering 속도와 플래시 빈도가 동적으로 증가해야 함. |
| **Pitch Deck / PPT** | 정적인 슬라이드, 제한된 애니메이션 기능. '시간의 흐름'을 극적으로 연출하는 것이 목표. | **애니메이션 시퀀스로 대체.** Red Zone 경고는 단순한 `Build-in` 트랜지션 효과로 구현하되, *글리치 필터와 노이즈 오버레이*를 배경 이미지/쉐이프로 삽입하여 '시스템 오류'의 느낌을 유지한다. |
| **Video (YouTube)** | 시간과 사운드 싱크가 가장 중요함. 90초 내 공포 곡선 설계 필수. | **사운드와 동기화된 비주얼.** 경고 문구 등장 시, 저음의 불안정한 험(Hum) 노이즈를 최고조로 올리고, 동시에 화면 떨림(Jittering)을 가장 강하게 적용하여 청각/시각적 공포를 통합한다. |
