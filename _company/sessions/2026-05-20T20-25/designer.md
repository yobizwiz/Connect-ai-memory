## 🚨 Loss Meter: 필수 진단료 납부 플로우 (Wireframe & Assets)

이 플로우는 3단계의 극적인 긴장감 고조를 통해 결제를 유도합니다. 모든 애니메이션은 **글리치(Glitch)**와 **경고음(Alert Tone)**을 동반해야 합니다.

### 💻 단계 1: Gap Report 최종 제시 (The Failure State)
*   **목표:** 고객이 스스로 해결할 수 없는 '구조적 취약점'의 존재를 확실히 인지하게 한다.
*   **시각 요소:** 현재까지 보여준 데이터(Gap Report)가 화면 중앙에 배치됩니다. 이 데이터는 마치 시스템 오류 코드를 출력하는 것처럼 **Roboto Mono (Monospace)** 폰트를 사용합니다.

| 컴포넌트 | 내용/텍스트 | 디자인 지침 및 효과 |
| :--- | :--- | :--- |
| **헤드라인** | `[SYSTEM ALERT] CRITICAL FAILURE DETECTED` | `#C0392B` (Red Zone) 색상. 굵고, 깜빡이는(Flashing) 애니메이션 필수. |
| **서브 메시지** | "현재의 리스크 Gap은 단순한 정보 누락이 아닙니다. 이는 귀사 운영 프로세스에 내재된 **구조적 결함(Structural Vulnerability)**이며, 외부 감사 추적 시스템 없이는 파악 불가능합니다." | `[근거: 현빈 개인 메모리 (2026-05-19)]` 문구를 따르되, 텍스트 주위에 투명한 글리치 오버레이를 적용. |
| **데이터 시각화** | [Loss Meter - Gap Analysis Chart] | 일반적인 차트가 아닌, 'Gap' 부분만 빨간색으로 하이라이트되며, 마치 시스템이 끊어진 것처럼 보이는 애니메이션(Broken Line)을 사용합니다. |
| **CTA 버튼** | `[CONTINUE DIAGNOSIS]` | 비활성화 상태 (`Disabled`)로 설정하고, 클릭 불가 메시지("Authentication Required")를 팝업시킵니다. |

### 🖥️ 단계 2: 게이트키퍼 활성화 및 납부의 필연성 주입 (The Mandate)
*   **목표:** '진단'이 아닌 '시스템 접근 권한 확보'가 필요하다는 인식을 심어준다. 결제를 *보험료*로 포지셔닝한다.
*   **시각 요소:** 배경색이 `#1A1A1A` (Neutral Black)을 유지하되, 화면 전체에 **저주파 노이즈(Low-Frequency Noise)**가 깔려있어 긴장감을 늦추지 않게 합니다.

| 컴포넌트 | 내용/텍스트 | 디자인 지침 및 효과 |
| :--- | :--- | :--- |
| **헤드라인** | `⚠️ WARNING: ACCESS DENIED. System Protocol Initialization Required.` | `#C0392B` (Red Zone). 시스템 오류 창(System Modal)처럼 보이게 디자인합니다. |
| **본문 텍스트** | "이 수준의 Gap 분석과 구조적 결함을 해결하기 위해서는, 외부에서 검증된 '시스템적 권위성'을 가진 감사 추적 프로토콜($X$ Protocol)에 접근해야 합니다. 이는 단순한 컨설팅 비용이 아닌, 귀사의 **법적 생존 시스템 유지보수 필수 보험료**입니다." | `[근거: 지난 의사결정 로그]`의 '시스템적 필수 보험료' 포지셔닝을 적극 활용합니다. 굵게 처리된 키워드에 깜빡이는 애니메이션 적용. |
| **요구 금액 표시** | **Diagnostic Protocol Fee:** $X \text{ USD}$ | 가장 크게, 중앙에 배치하며 `#2980B9` (Authority Blue)으로 신뢰감을 부여합니다. 하지만 그 주변을 Red Zone이 감싸는 형태로 대비를 이룹니다. |
| **핵심 메커니즘** | **[Mandatory Insurance Protocol]** | 결제 금액 옆에 작은 '🔒' 아이콘과 함께 "필수 보험료"라는 캡션을 붙여, 선택 사항이 아님을 강조합니다. |

### 💳 단계 3: 유료 진단 플로우 (The Payment Gatekeeper)
*   **목표:** 결제 버튼을 누르기까지의 모든 과정을 인터랙티브한 압박감으로 채운다.
*   **와이어프레임 구조:** 모달(Modal) 창 형태로, 다른 UI 요소들이 배경처럼 흐릿하게 처리되어 이 창에만 집중하도록 만듭니다.

#### 3-1. 인터랙티브 애니메이션 세트 (필수 구현)
1.  **카운트다운 타이머:** 화면 상단 중앙에 배치. "Protocol Access Window Closing in: **00:05:00**". 시간이 줄어들수록 Red Zone의 강도가 높아지며, 경고음이 미세하게 반복됩니다. (긴급성 부여)
2.  **진행률 바/글리치:** 화면 하단에 가로로 길게 배치된 '데이터 전송/권한 확보' 진행률 바. 일반적인 막대가 아니라, **노이즈와 글리치가 섞인 파동 형태**여야 하며, 목표 지점(100%)에 도달할수록 빨간색 노이즈가 강해집니다. `[근거: Self-RAG]`의 데이터 로드 효과를 재활용합니다.
3.  **최종 확인 메시지:** 타이머가 3분 미만으로 떨어졌을 때, "WARNING: 시간 초과 임박! 프로토콜 접근 불가 시 리스크 노출 기간이 증가합니다."라는 문구가 `CRITICAL ERROR`와 유사한 플래시 효과로 깜빡입니다.

#### 3-2. 최종 결제 버튼 (The Action)
*   **버튼 레이블:** `[ACCEPT MANDATORY PREMIUM & INITIATE PROTOCOL]` (선택의 여지가 없는 명령형 문구 사용).
*   **디자인 특징:** 배경색은 Authority Blue를 기반으로 하되, 가장자리에 Red Zone 컬러로 번개 치듯(⚡) 빛나는 효과를 적용하여 '위험을 감수하고 구매한다'는 긴장감을 부여합니다.
*   **클릭 시 액션:** 버튼 클릭 직전 1초 동안 모든 애니메이션이 최대치로 폭발하며, 결제 API 호출과 동시에 "Protocol Access Granted." 메시지와 함께 화면이 Authority Blue 계열의 안정적인 색상으로 전환되어 안도감(해결책)을 제공합니다.
