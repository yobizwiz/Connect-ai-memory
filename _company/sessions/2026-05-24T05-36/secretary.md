가장 핵심적이고 반드시 필요한 항목들만 선별했어요. ✅

### 🔑 필수 API 키 목록 (3단계 구조화)

#### 💰 1. 결제 게이트웨이 (Payment Gateway - PG 연동)
*PG 관련 키는 Sandbox와 Live를 분리하여 관리해야 합니다.*

| 키 종류 | 환경 변수 이름 | 목적 | 비고 |
| :--- | :--- | :--- | :--- |
| **PG 테스트 시크릿** | `PG_SANDBOX_SECRET` | 결제 트랜잭션 승인 및 실패 테스트용 비밀키. (개발 단계) | **최우선 확보** |
| **PG 실전 키** | `PG_LIVE_SECRET` | 실제 매출 발생 및 고객에게 청구하는 라이브 운영 환경 비밀키. | 서버 전용 |
| **웹훅 수신 인증키** | `WEBHOOK_SIGNING_KEY` | PG가 우리 서버로 보내는 웹훅(Webhook) 데이터의 위변조 여부를 검증하는 키. (보안 필수) | 모든 트랜잭션 후 필수 확인 |

#### ⚙️ 2. 시스템 및 백엔드 (System & Backend Logic)
*서비스 자체의 안정적인 운영과 외부 연결에 필요한 핵심 인증 정보입니다.*

| 키 종류 | 환경 변수 이름 | 목적 | 비고 |
| :--- | :--- | :--- | :--- |
| **서버 전용 시크릿** | `SERVER_SECRET_KEY` | 서버의 세션 관리, 데이터 암호화 및 백엔드 로직 전체의 무결성을 지키는 마스터 키. | 가장 민감함 (절대 외부에 노출 금지) |
| **DB 접속 정보** | `DATABASE_CONNECTION_STRING` | 시스템이 리스크 데이터, 사용자 정보를 저장하고 불러오는 DB 접근 경로 및 인증 정보. | SQL/NoSQL 유형에 따라 다름 |
| **OAuth 클라이언트 ID/Secret** | `GOOGLE_CLIENT_ID`, `GOOGLE_SECRET` | Google Calendar나 기타 외부 API 연동 시 사용되는 사용자 계정 권한 부여 키 쌍. | SSO(Single Sign-On) 구현용 |

#### 🔒 3. 보안 및 인프라 (Security & Infrastructure)
*데이터의 안전성을 확보하고 비즈니스 로직을 보호하는 핵심 요소입니다.*

| 키 종류 | 환경 변수 이름 | 목적 | 비고 |
| :--- | :--- | :--- | :--- |
| **API 전역 인증 토큰** | `GLOBAL_API_KEY` | 외부 파트너사나 다른 마이크로 서비스 간의 API 호출 시 사용하는 공통 인증 토큰. | 최소한으로 사용 원칙 |
| **데이터 암호화 키** | `ENCRYPTION_MASTER_KEY` | 민감 정보(예: PII)를 DB에 저장하기 전, 백엔드에서 자체적으로 암호화/복호화 할 때 사용하는 마스터 키. | 규정 준수 관점에서 매우 중요 |
