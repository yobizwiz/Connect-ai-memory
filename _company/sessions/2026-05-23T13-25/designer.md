## 🚨 위협 지수 시각화 시스템 (Threat Index Visualization System) 스펙 v1.0

### 🎯 1. 핵심 디자인 원칙 및 아키텍처
*   **목표:** 고객의 '시스템적 무지(Systemic Blindness)'를 자극하여, 현재 상태가 위험하며 즉각적인 진단이 필요하다는 긴급성을 각인시킨다. [근거: Designer 메모리]
*   **톤앤매너:** 시스템 경고 (System Alert), 권위적 오류 메시지 (High-Authority Error Message).
*   **핵심 구조:** **낮은 위협(Green/Blue) $\rightarrow$ 임계점 근접(Yellow/Orange) $\rightarrow$ 치명적 위험(Red Zone)**의 3단계 컬러/효과 변화를 강제한다. [근거: Self-RAG]

### 🎨 2. 색상 및 타이포그래피 가이드라인 (재확인)
| 역할 | 색상 코드 (HEX) | 사용 목적 및 효과 | 적용 미디어 |
| :--- | :--- | :--- | :--- |
| **🔵 Authority Blue** (안전/해결책) | `#2980B9` | 시스템의 근거, 권위적 진술. 안정적인 배경 텍스트. [근거: Self-RAG] | 전 채널 |
| **🟡 Warning Yellow** (경고 임계점) | `#FFC300` | 주의 필요 구간. 불안정성의 시작을 알리는 색상. [추측 - 긴장감 유발] | 모든 시각화 섹션 |
| **🚨 Red Zone** (위협/공포) | `#C0392B` | 치명적 위험, 구조적 취약점 발견 시. 페이지를 압도하는 충격. [근거: Self-RAG] | 전 채널 (가장 중요) |
| **⚫ Neutral Black** (기본 배경) | `#1A1A1A` | 기본 다크 모드 배경. 전문성과 깊이감 유지. [근거: Self-RAG] | 전 채널 |

*   **폰트:** 메인 텍스트는 `Inter` (가독성), 데이터/경고 코드는 `Roboto Mono` (시스템 코드 느낌)를 반드시 분리하여 사용한다. [근거: Self-RAG]

### 🖼️ 3. 시각화 시스템 스토어보드 및 애니메이션 스펙
위협 지수는 **[Threat Score %]** 라는 수치로 표현하며, 이 수치가 변할 때마다 다음의 애니메이션 트랜지션이 발생해야 합니다.

#### A. [상태: 정상/저위험 (0% $\rightarrow$ 70%)] - Authority State
*   **UI:** 위협 지수 게이지가 화면 하단에 고정된 바(Bar) 형태로 존재한다. 색상은 `🔵 Authority Blue`에서 점진적으로 `🟡 Warning Yellow`로 변화하는 그라디언트 효과를 준다.
*   **애니메이션:** 데이터 업데이트 시 부드러운 리액션(Smooth Interpolation). 마치 살아있는 그래프처럼 보이지만, '안정적'이라는 인상을 줘야 한다. (속도: 느리고 규칙적)

#### B. [상태: 경고 임계점 진입 (70% $\rightarrow$ 90%)] - Pre-Alert State
*   **트리거:** 위협 지수가 70%를 초과하는 순간.
*   **시각 효과:**
    1.  **색상 변화:** 게이지 색상이 `🟡 Warning Yellow`로 급격히 변하고, 배경에 낮은 주파수의 **'잔류 노이즈(Residual Noise)'** 오버레이가 겹쳐지기 시작한다. (불안정성의 초기 신호)
    2.  **사운드:** 규칙적이던 백그라운드 BGM의 피치가 아주 미세하게 불안정한 떨림(Pitch Flutter)을 추가한다.
    3.  **UI 반응:** '⚠️ 경고: 데이터 불일치'라는 문구가 `Roboto Mono` 폰트와 함께 화면 중앙 하단에 깜빡임 애니메이션으로 노출된다.

#### C. [상태: Red Zone 진입 (90% $\rightarrow$ 100%)] - CRITICAL ERROR State
*   **트리거:** 위협 지수가 90%를 초과하는 순간 **(가장 중요)**. 이는 스토리의 클라이맥스이며, 시스템이 '오류'를 발생시킨 것처럼 연출해야 한다.
*   **애니메이션/충격 효과 (개발자 지침):**
    1.  **플래시:** 화면 전체에 200ms 동안 강하고 빨간색의 **`#C0392B` 플래시**가 지나간다. `Intersection Observer`를 활용하여, 이 섹션에 도달하는 순간 트리거되도록 한다.
    2.  **글리치 노이즈:** 배경 전체에 낮은 빈도의 노이즈(Noise)와 **색상 왜곡(Chromatic Aberration)** 필터를 강제로 오버레이한다. (CSS Filter 적용 필수). 이 효과는 화면의 모든 요소(텍스트, 이미지, UI 등)를 뒤틀어 보이게 만든다. [근거: Self-RAG]
    3.  **메시지 폭발:** 'CRITICAL ERROR' 메시지가 폰트 크기 변화와 함께 `text-shadow` (빨간색과 파란색의 떨리는 그림자) 효과를 주며 화면 중앙에 강하게 고정된다.
    4.  **오디오:** 높은 주파수의 날카로운 '경고음(Alarm Sound)'을 1초간 재생하고, 비트가 깨지는 듯한 `Glitch SFX`를 추가한다.

### 🌐 4. 미디어별 적용 방안 (Implementation Strategy)

| 채널 | 구현 방식 | 특이 사항 및 주의점 |
| :--- | :--- | :--- |
| **유튜브/랜딩 페이지** | 웹/영상 전역 애니메이션 오버레이 (CSS Filter + JS Trigger) | Red Zone 진입 시, 모든 요소가 '시스템 오류'처럼 보이도록 연출해야 한다. 화면 전체를 압도하는 경험이 중요함. |
| **인스타그램 릴스** | 비디오 에디터의 효과/색상 보정 필터 및 오버레이 (Overlay) 활용. | Red Zone 구간에서 인위적으로 대비되는 강한 플래시와 노이즈(Noise)를 삽입한다. 글리치 효과는 '텍스트 왜곡'에 집중하고, 배경은 단순하게 유지하여 메시지 전달력을 높인다. |
| **썸네일** | 3단계 컬러 팔레트의 조합을 직관적으로 보여주는 레이아웃 설계. | 썸네일 자체에 Red Zone 느낌을 최소한으로 포함시켜야 한다 (예: 경고 아이콘, 빨간색 테두리). 너무 많은 노이즈는 오히려 클릭률을 낮출 수 있으니 주의한다. [근거: Designer 메모리] |
