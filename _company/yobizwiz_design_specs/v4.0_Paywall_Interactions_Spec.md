# 🚨 YOBIZWIZ: Paywall & Red Zone Alert Interaction Protocol (V4.0)

**목표:** '공포(Fear)' $\rightarrow$ '위협 인식(Threat)' $\rightarrow$ '해결책 필요성(Authority)'을 극대화하여, 최종적으로 결제 버튼(CTA) 클릭을 강제한다. 모든 애니메이션은 엔지니어링 관점에서 구현 가능해야 하며, 60 FPS를 목표로 한다.

---

## I. 핵심 컴포넌트별 인터랙티브 스펙 (The Protocol)

### 1. Red Zone Alert: Lmax 수치 급증 경고 시스템 (Fear State Trigger)
*   **발동 조건:** 리스크 점수 게이지가 임계치를 초과하고, $L_{max}$ 값이 이전 프레임 대비 **50% 이상** 급증할 때 발동.
*   **시각적 목표:** 시스템의 오류와 붕괴를 느끼게 한다 (Panic/Urgency).
*   **애니메이션 로직 (3단계 순차 트리거):**

    1.  **[Lmax Jump] 스파이크 효과 (0ms - 200ms):**
        *   $L_{max}$ 수치가 일반적인 `Roboto Mono` 폰트를 벗어나, 순간적으로 **글리치 노이즈(Glitch Noise)**를 동반하며 점프한다.
        *   효과: X축/Y축으로 짧게 진동(`transform: translate3d(0px, -5px, 0)`)하고, 수치 주변에 네온 레드 플래시 오버레이가 깜빡인다. (Opacity 100% $\rightarrow$ 20% $\rightarrow$ 100%).
        *   **색상:** `#C0392B` (Dark Crimson).
    2.  **[System Alert] 경고 아웃라인 (200ms - 500ms):**
        *   경고 박스 전체에 **지터링(Jittering)** 효과가 걸린다. 이는 배경 전체에 낮은 빈도의 `Noise/Chromatic Aberration` 필터 오버레이를 강화하는 방식으로 구현한다.
        *   효과: 마치 데이터 패킷이 불안정하게 전송되는 듯한 시각적 왜곡을 부여한다. (CSS Filter 적용).
    3.  **[Audio/Haptic Cue] (동시):** 강렬하고 짧은 경고음(Stuttering Beep)과 함께 화면 전체에 붉은색 잔상이 남는다.

### 2. Silver Tier Solution Area: 안정감 구축 영역 (Authority State Anchor)
*   **발동 조건:** 사용자가 '솔루션 탐색' 섹션으로 스크롤하거나, 리스크 경고가 최고조에 달한 후 잠시 정지할 때 발동.
*   **시각적 목표:** 공포를 신뢰(Trust)로 전환시키고, 해결책의 우아함을 강조한다 (Resolution/Calm).
*   **애니메이션 로직 (3단계 순차 트리거):**

    1.  **[Dimensional Lift] 스테이트 변화 (0ms - 300ms):**
        *   이 섹션은 주변 Red Zone과 확실히 분리되어야 한다. 마치 시스템의 핵심 모듈처럼 보이게, **미세한 광원 효과(Inner Glow)**와 `Glassmorphism` 테두리가 입체감 있게 떠오르는 듯한 애니메이션을 적용한다.
        *   효과: 배경색 대비를 극대화하고, 경계가 명확하게 정의되어야 한다. (Depth Perception 유도).
    2.  **[Data Flow] 권위 데이터 스트리밍 (300ms - 지속):**
        *   솔루션의 기능 설명(예: '법적 컴플라이언스', '자동화된 감사 로깅')이 나열될 때, 각 항목 옆에 작은 **Authority Blue (`#2980B9`)** 색상의 미세한 데이터 흐름 애니메이션 (점-선 연결)을 적용하여 기술적인 전문성을 시각적으로 증명한다.
        *   애니메이션: `SVG Path` 기반의 부드러운 트래킹 효과를 사용한다.

### 3. 최종 CTA 버튼: Action Gate (Conversion Point)
*   **발동 조건:** 모든 공포와 권위가 최고조에 달한 후, 마지막 결제 유도 순간.
*   **시각적 목표:** 시선을 강제로 집중시키고, 클릭을 '의무'처럼 느끼게 만든다.
*   **애니메이션 로직 (2단계 순차 트리거):**

    1.  **[Pre-Click Pulse] 광원 강조 (0ms - 500ms):**
        *   버튼 자체에 Authority Blue를 기반으로 하는 **펄스(Pulse)** 효과를 주기적으로 반복한다. 이는 단순히 크기 변화가 아닌, 버튼 테두리 전체에 빛이 퍼져나가는 듯한 `box-shadow` 애니메이션을 사용해야 한다.
        *   **지연:** 1.5초 간격으로 강도 높은 광원 깜빡임을 유지하여 무의식적으로 주목하게 만든다.
    2.  **[Hover/Click Feedback] 인터랙티브 피드백 (On Interaction):**
        *   **Hover:** 마우스 커서가 근접할 때, 버튼이 미세하게 위로 떠오르는(Lift) 효과와 함께 배경색이 10% 더 밝게 채도 높은 청색으로 변한다. (`translateY(-2px)`).
        *   **Click:** 클릭 순간에는 모든 애니메이션을 일시 정지시키고, 버튼 전체가 짧은 시간 동안 **압축되는(Compression)** 효과를 주어 물리적인 '결정'의 느낌을 부여한다.
