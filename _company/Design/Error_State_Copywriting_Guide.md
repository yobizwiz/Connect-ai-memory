# 🚨 시스템 에러 페이지 카피라이팅 가이드 (v1.0)

## 🎯 목표: 결제 실패를 '구조적 리스크 경고'로 재포장
*   **원칙:** 기술적인 오류 코드나 카드사 문제는 언급하지 않습니다. 오직 **'당신의 비즈니스에 존재하는 법규 준수 Gap(D 체크)'**만을 이야기합니다.
*   **톤앤매너:** 권위적, 위협적, 시스템 보고서 스타일 (Diagnostic Report).

## 💡 사용 시나리오별 카피 선택

| 상황 | 추천 옵션 | 핵심 메시지 전환 | CTA 목표 |
| :--- | :--- | :--- | :--- |
| **[1] 강한 긴급성 유도** (최적) | Option 1 🔥 | "결제 실패 = 리스크 노출 증거" | 즉시 재시도(Retry Payment) |
| **[2] 전문성 강조** (차선) | Option 2 🛡️ | "시스템 오류 = 진단 미완료의 경고" | 재진단 세션 재개(Retry Diagnosis) |
| **[3] 이탈 방지/재접근** | Option 3 🤝 | "오류 발생 = 전문가 상담 필요" | 관리자 문의 및 일정 예약(Book Call) |

## ✨ 핵심 구성 요소 (System Copy Block)

### 1. 헤드라인 (H1):
`[CRITICAL SYSTEM WARNING] - 데이터 무결성 위협 감지.`

### 2. 본문 리스크 경고 메시지:
"지금 발생한 오류는 단순한 결제 문제가 아닙니다. 이는 당신의 핵심 프로세스에 **'미확인된 구조적 리스크 공백(Structural Gap)'**이 존재한다는 강력한 증거입니다. 이 공백을 방치하는 것은 곧 법규 준수 위반과 재무적 손실로 이어질 수 있습니다."

### 3. 다음 단계 안내 (CTA):
"진단은 중단할 수 없습니다. 리스크를 즉시 봉쇄하기 위해, 결제 정보를 확인하고 **최종 진단 세션**을 완료하십시오."
