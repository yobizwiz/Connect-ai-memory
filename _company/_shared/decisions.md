## 🏛️ 핵심 전략 (확정 — 모든 에이전트 필수 참조)

### 1. 회사 정체성
- **회사명**: YobizWiz — 1인 기업 AI OS
- **핵심 포지셔닝**: 단순 솔루션 판매가 아닌, '구조적 위협 방어 보험료(Premium)' 판매
- **핵심 가치**: 법규 준수가 아닌 **절차적 무결성(Procedural Integrity)** 증명과 **구조적 취약성(Structural Vulnerability)** 진단
- **타겟 고객**: B2B 전문직 (법률, 금융, 의료 등) — 고가치 컨설팅 대상

### 2. 수익 모델 (확정)
- **3단계 퍼널**: 무료 진단 → 유료 컨설팅(ACV) → 연간 구독(MRR)
- **가격 구조**: 3단계 티어 ($199/월 Bronze ~ Silver ~ Gold)
- **전환 트리거**: 무료 보고서의 '불완전성(Scope/Action/Scale 제한)' 고지 → 유료 전환
- **Anti-Exit Gateway**: CTA 직전에 리스크 레벨 상승 시각화로 이탈 방지

### 3. 마케팅 원칙 (확정)
- **톤앤매너**: 권위적 긴급성 — '존재론적 위협(Existential Threat)' 경고
- **핵심 메시지**: 기능 소개 X → 구체적 재무 손실액($10M~$50M) 수치화
- **시각 언어**: Red Zone(`#C0392B`) + Authority Blue 대비, 글리치/노이즈 효과 필수
- **최종 CTA**: 모든 콘텐츠의 최종 목표 = 유료 리스크 진단 체험 요청
- **스토리 구조**: 위협 체감(Red Zone) → 전문성 입증 → 행동 촉구(3단계)

### 4. 기술 아키텍처 (확정)
- **백엔드**: FastAPI + 리스크 엔진(독립 서비스 레이어)
- **API**: POST /api/v1/risk-assessment, Rate Limiting 필수, JSON Schema 엄수
- **리스크 계산**: 업종 가중치 반영, QLoss(잠재적 손실) 수치화, 감사 추적(Audit Trail) 핵심
- **결제**: Stripe SDK 통합 (Mocking → 실제 PG 전환 완료)
- **Loss Meter**: 인터랙티브 프로토타입, Pre-Checkout 통합, Critical Alert 시 글리치 효과
- **데이터 보안**: PII 마스킹, OAuth 2.0 필수, 환경변수/Key Vault로 민감정보 관리

### 5. 콘텐츠 전략 (확정)
- **유튜브**: 후킹 분석기 스킬팩 장착, 트래픽 유입 목적
- **인스타그램**: Red Zone 컬러 기반 '긴급 시스템 경고' 릴스 제작
- **레딧**: 프로토타입 완성 후 커뮤니티 피드백 수집
- **랜딩 페이지**: 정보 제공 X → '작동하는 시뮬레이터 시스템'으로 구현

### 6. UX/UI 원칙 (확정)
- **다크 모드 배경** + Red Zone 경고 → Authority Blue 해결책 전환
- **결제 직전**: 3초 지연 + 다단계 상태 전환 + 글리치 효과로 불안감 극대화
- **Loss Meter**: 초기 상태 = Red Zone, 사용자 변수 기반 불안정성 시뮬레이션
- **모든 인터랙션 지점**: Tracking Hooks(추적 API) 필수 구현
