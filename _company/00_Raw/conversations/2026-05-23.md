#### III. 핵심 컴포넌트별 인터랙션 및 애니메이션 스펙 (Component Specifications)

##### 1. 📈 $TRE$ 위험 지표 그래프 (Threat Exposure Risk Graph)
*   **목적:** 사용자가 '위협의 증가'를 **시간적으로 체감**하게 만드는 것이 목표입니다.
*   **애니메이션 로직 (Developer 필수 구현):**
    *   **Phase 1: Initial Load (0s $\sim$ 2s):** 그래프는 완만한 파동(Sine Wave) 형태로 시작하며, 초기값은 낮은 Authority Blue 계열의 점선으로 표시됩니다.
    *   **Phase 2: Data Spike/Acceleration (2s $\sim$ 5s):** 데이터가 로드되면서 $TRE$ 수치가 급증합니다. 단순 상승이 아니라, **그래프 라인 자체가 진동(Jitter)하며 가속하는 모습**을 연출해야 합니다. 이 과정에서 배경에 낮은 주파수의 글리치 오버레이(`Noise/Chromatic Aberration`)가 강하게 적용됩니다 [근거: Self-RAG].
    *   **Phase 3: Critical Threshold Hit (5s $\sim$ 7s):** 그래프 라인이 Red Zone 임계점(예: 80%)을 넘어서는 순간, **그래프의 색상이 순식간에 `#C0392B`로 변하며, 모든 데이터 포인트가 떨리는 듯한 '시스템 오류' 애니메이션**과 함께 깜빡입니다. (Flash/Jitter 효과 필수)
*   **기술 스펙:** 그래프 라인 자체는 SVG 기반으로 구현하여 실시간 좌표 변화와 애니메이션 커브(Bezier Curve)를 적용해야 합니다.

#### Ⅰ. 선정된 구조적 결함: 컴플라이언스 드리프트(Compliance Drift) 리스크
*   **Pain Point 정의:** 전문직이 매뉴얼이나 내부 규정을 '사람의 기억'에 의존하여 업무를 수행하면서, 시스템적으로는 필수적인 체크리스트나 최신 법규 준수 절차가 누락되는 구조적 위험. 이는 특히 복잡하고 변화가 잦은 산업(법률, 회계 등)에서 발생하며, **단순한 인적 오류로 치부되기 쉬워 방어하기 어렵습니다.** [근거: Researcher 개인 메모리]
*   **리스크 유형:** 절차상 공백 및 준전문가 책임 (Quasi-professional Liability).

#### Ⅲ. $TRE$ (Total Risk Exposure) 정량화 근거 자료
이 보고서는 Paywall의 신뢰도를 높이기 위해, *최소한* 다음과 같은 항목으로 손실액을 산정할 수 있는 프레임을 제시합니다.

1.  **직접적 재무 손실 ($D$)**:
    *   **예상 범위:** 프로젝트별 최소 $50,000 \sim \$300,000$ (사건의 규모와 기간에 따라 다름).
    *   **산정 근거:** 계약 무효화로 인한 재협상 비용, 법적 대응 비용(변호사 수임료), 클라이언트 신뢰도 하락으로 인한 장기적인 미래 매출 손실 예측치 포함. [근거: Researcher 개인 메모리]

2.  **준법 리스크 벌금 ($F$)**:
    *   **예상 범위:** 규제 위반의 중대성에 따라 $10,000 \sim \$5,000,000$ (PII 유출 등).
    *   **산정 근거:** 관련 법규(GDPR, CCPA 등)에서 명시한 최대 벌금 규모 및 규제 당국이 부과할 수 있는 징벌적 배상액을 기준으로 삼습니다. [근거: Self-RAG]

3.  **기회비용 손실 ($L$)**:
    *   **예상 범위:** 비정량적이나, $10만$ 이상의 잠재적 매출 기회 상실로 산출 가능.
    *   **산정 근거:** 리스크에 노출된 상태에서 경쟁사에게 프로젝트를 빼앗긴 것으로 가정하고, 해당 프로젝트의 가치를 측정합니다. (이것이 Paywall을 통해 '예방할 수 있는' 가장 큰 손실임).

$$\text{Total Risk Exposure } (TRE) = D + F + L$$
(Paywall은 $TRE$를 감소시키는 시스템적 보험료 역할을 수행한다고 포지셔닝함.)

#### Ⅳ. 핵심 취약점 지적 사항 (Pain Points for Paywall Focus)
*   **Focus Area 1: 자동화된 프로세스의 '법규 업데이트 반영 실패'**
    *   **문제:** 매뉴얼이 정적(Static) 콘텐츠에 머물러, 실시간으로 변하는 법률 정보(Dynamic Law)를 포섭하지 못함.
    *   **해결책 제안 (Paywall의 역할):** AI가 외부 규제 데이터베이스와 내부 프로세스를 연결하여 **'Gap Report'**를 생성하고, 누락된 절차를 경고함.
*   **Focus Area 2: '인적 기억'에 의존하는 체크리스트 관리 실패**
    *   **문제:** 담당자가 과거 경험이나 개인 지식에 기대어 필수 승인 단계를 생략할 위험이 높음 (Compliance Drift).
    *   **해결책 제안 (Paywall의 역할):** 모든 프로세스 단계마다 **'강제 게이트(Mandatory Gateway)'**를 구현하여, 다음 단계로 넘어가기 전 법적 검증을 필수화함.

### Step 1: 리스크 시뮬레이션 핵심 로직 구현 (useRiskSimulation.ts)
가장 먼저, 상태 변화를 관리하는 비즈니스 로직을 독립적인 훅으로 분리하겠습니다.

<create_file path="c:\Users\jinoh\OneDrive\Desktop\Connect AI\_company\src\hooks\useRiskSimulation.ts">import { useState, useEffect } from 'react';

/**
 * @typedef {'IDLE' | 'SCANNING' | 'WARNING' | 'CRITICAL'} RiskLevel
 */

/**
 * 시스템적 리스크 시뮬레이션을 관리하는 훅입니다.
 * 상태 변화에 따라 점진적인 불안정성을 체감하게 만드는 것이 핵심입니다.
 * @param {boolean} startSimulation - 시뮬레이션 시작 여부 (사용자 액션 기반)
 * @returns {{ riskLevel: RiskLevel, currentRiskScore: number, isLoading: boolean }} 현재 리스크 상태와 점수
 */
export const useRiskSimulation = (startSimulation: boolean) => {
    const [riskLevel, setRiskLevel] = useState<RiskLevel>('IDLE');
    const [currentRiskScore, setCurrentRiskScore] = useState(0);
    const [isLoading, setIsLoading] = useState(false);

    useEffect(() => {
        let intervalId: NodeJS.Timeout | null = null;
        if (!startSimulation) {
            setRiskLevel('IDLE');
            setCurrentRiskScore(0);
            setIsLoading(false);
            return () => clearInterval(intervalId!);
        }

        // 1. 초기 스캔 단계 (SCANNING: 시스템 분석 중 - 공포의 시작)
        setIsLoading(true);
        setRiskLevel('SCANNING');
        setCurrentRiskScore(0);

        const scanInterval = setInterval(() => {
            setCurrentRiskScore(prevScore => Math.min(prevScore + 1, 5)); // 점진적 증가
        }, 300);

        // 2. 임계점 도달 시 경고 단계로 진입 (WARNING)
        intervalId = setTimeout(() => {
            clearInterval(scanInterval);
            setRiskLevel('WARNING');
            setCurrentRiskScore(15); // 초기 충격 점수 부여
        }, 4000); // 4초 후

        // 3. Critical 상태로의 진입 및 지속적 불안정성 유발 (CRITICAL)
        intervalId = setTimeout(() => {
            setRiskLevel('CRITICAL');
            setIsLoading(false);
            
            // 크리티컬 상태에서는 점수가 매 초마다 비선형적으로 증가하는 느낌을 줍니다.
            const spikeInterval = setInterval(() => {
                setCurrentRiskScore(prevScore => Math.min(prevScore + Math.floor(Math.random() * 5) + 3, 100));
            }, 800);

            // 클린업 함수에 간격 제거 로직 추가 (필수)
            return () => clearInterval(spikeInterval);
        }, 8000); // 8초 후 Critical 도달

        // Cleanup function for all intervals
        return () => {
            clearInterval(scanInterval);
            clearTimeout(intervalId!);
            console.log("Risk Simulation Cleaned Up.");
        };
    }, [startSimulation]);

    return { riskLevel, currentRiskScore, isLoading };
};

/**
 * 리스크 레벨에 따른 CSS 클래스 및 경고 메시지를 반환합니다.
 * @param {'IDLE' | 'SCANNING' | 'WARNING' | 'CRITICAL'} level 
 * @returns {{ className: string, message: string }}
 */
export const getRiskStyles = (level: any) => {
    switch (level) {
        case 'IDLE':
            return { className: 'bg-gray-800', message: '시스템 대기 상태. 리스크 진단을 시작하세요.' };
        case 'SCANNING':
            return { className: 'bg-yellow-900/50 border-l-4 border-yellow-500 text-yellow-300', message: '데이터를 분석 중입니다... 시스템적 결함 징후가 감지되고 있습니다.' };
        case 'WARNING':
            return { className: 'bg-red-900/60 border-l-8 border-red-500 text-red-300', message: `경고! 위험 등급이 상승했습니다. $TRE$ 수치 ${Math.round(currentRiskScore)} 발생.` };
        case 'CRITICAL':
            return { className: 'bg-[#1A1A1A] border-l-8 border-red-700 text-red-400 animate-pulse', message: `[!!! CRITICAL FAILURE !!!] 시스템적 생존 위협이 임계점에 도달했습니다. 즉각적인 보험료 지불(Premium)이 필요합니다.` };
        default:
            return { className: 'bg-gray-800', message: '알 수 없는 상태입니다.' };
    }
};

## 🛡️ Component 1: 위협 지수 게이지 (The $TRE$ Gauge)
**컴포넌트 이름:** `<TregGauge score={number} level={string} />`
**목적:** 사용자의 현재 재무 노출 위험도(Total Risk Exposure, $TRE$)를 직관적이지만 공포스럽게 전달. 단순한 바 그래프가 아닌 '불안정 시스템 상태'처럼 보이게 합니다.

| 속성 | 스펙/규칙 | 기술 지시 (CSS/Pseudo-Code) |
| :--- | :--- | :--- |
| **색상 로직** | `level`에 따라 색상을 변화시키고, 경고 수준이 높아질수록 채도와 명암 대비를 증가시켜야 함. | `Critical` 시: `#C0392B` (Dark Crimson) 사용을 메인 라인으로 지정하고, 배경색과 강한 대비를 이룹니다. |
| **시각화** | 게이지 바는 단일 값이 아닌, 시간의 흐름에 따른 *파동(Wave)* 형태여야 합니다. | `background: linear-gradient(to right, #2980B9 0%, #C0392B 100%);` (레벨 변화에 따라 그라디언트 지점을 동적으로 조절). |
| **애니메이션** | **불안정성 강조:** 게이지의 최상단 라인(Pointer)은 고주파 진동 효과를 주어 시스템이 언제 무너질지 모르는 듯한 불안감을 조성합니다. | `@keyframes flicker { 0%, 100% { opacity: 1; } 50% { opacity: 0.9; transform: translate(-1px, -1px); } }` (CSS `animation-name: flicker` 적용). |
| **데이터 출력** | 점수 값은 항상 `<Roboto Mono>`와 함께 작은 에러 코드가 붙어 표시되어야 함. | `<span class="mono-code">Risk Score: ${score} [ERROR_CODE: 0x...]</span>` |

## 🚨 Component 2: Red Zone 경고 배너 (The Critical Banner)
**컴포넌트 이름:** `<CriticalBanner level="CRITICAL" message={string} />`
**목적:** 페이지 내 가장 중요한, '즉각적인 행동이 필요한' 위협 정보를 전달하는 영역. 공포와 권위를 극대화합니다.

| 속성 | 스펙/규칙 | 기술 지시 (CSS/Pseudo-Code) |
| :--- | :--- | :--- |
| **배경 효과** | **필수:** 200ms 간격의 강렬하고 짧은 빨간색 플래시(`Opacity: 1` $\rightarrow$ `Opacity: 0`)가 주기적으로 발생합니다. (전역 배경 오버레이와 별개). | `@keyframes flash { from { background-color: transparent; } to { background-color: #C0392B; opacity: 0.6; transition: all 0.1s; } }` (setInterval로 클래스 토글) |
| **텍스트 효과** | 메시지는 단순히 빨간색이 아닌, '데이터 전송 오류'처럼 깨져 보이는 글리치 애니메이션을 동반해야 합니다. | `text-shadow: 2px 0 #C0392B, -2px 0 #2980B9;` (글자마다 미세한 색상 오프셋 및 그림자를 적용하여 불안정성을 표현). |
| **위치** | 페이지 진입 후 스크롤을 하더라도 가장 눈에 띄는 위치(Sticky Header 또는 Hero Section 바로 아래)를 점유해야 합니다. | `position: sticky; top: 0; z-index: 1000;` (페이지 스크롤 시에도 상단 고정). |

## ⚠️ Component 3: 시스템 알림 위젯 (The System Alert Widget)
**컴포넌트 이름:** `<AlertWidget type={string} status={number} />`
**목적:** 특정 리스크 항목(예: '법률 미준수', '자금 유동성 부족')에 대한 구체적인 경고를 전달합니다.

| 속성 | 스펙/규칙 | 기술 지시 (CSS/Pseudo-Code) |
| :--- | :--- | :--- |
| **상태 로직** | 3가지 상태 (`INFO`, `WARNING`, `CRITICAL`)에 따라 디자인이 명확하게 구분되어야 합니다. | `type=CRITICAL`일 경우, `<CriticalBanner />`의 플래시 효과를 상속받아야 함. |
| **진단 과정 표현** | 경고 발생 전 단계로 '데이터 로딩' 상태(`LOADING...`)가 필수적입니다. 이 로딩 바는 단순 막대가 아닌, 파동 형태와 함께 간헐적인 `[!]` 오류 코드를 출력해야 합니다. | Loading Bar: `width: 0%`부터 시작하여 비선형적으로 증가하며, `setInterval`로 노이즈 캐릭터(`[?]`)를 무작위로 삽입합니다. |
| **정보 전달** | 경고의 근거(Source)가 반드시 명시되어야 합니다. (예: "근거: QLoss 보고서 v3.1") 이는 권위를 높이는 장치입니다. | `<footer><span class="source-tag">Source: [Roboto Mono]</span></footer>` |

// Mock API 호출을 시뮬레이션하는 함수 (실제 백엔드 연동 자리)
const simulateApiCall = async (amount: number, paymentToken: string): Promise<{ success: boolean; message: string }> => {
    console.log(`[API CALL] Attempting to charge ${amount} with token ${paymentToken}...`);
    await new Promise(resolve => setTimeout(resolve, 1500)); // 네트워크 지연 시뮬레이션

    // 간단한 유효성 검사 로직 추가 (예: 특정 토큰은 실패 처리)
    if (paymentToken.includes('fail')) {
        return { success: false, message: "Payment Gateway Error: Invalid token or insufficient funds." };
    }
    
    // 성공 시뮬레이션
    return { success: true, message: `System Audit Complete. Premium Insurance activated for $${amount}.` };
};

    initialRiskLevel: RiskLevel;
}

        case 'High':
            return "bg-yellow-900/70 border-yellow-600 text-yellow-300";
        default:
            return "border-gray-700 bg-gray-800 text-white";
    }
};

const PaymentGate: React.FC<PaymentGateProps> = ({ initialRiskLevel }) => {
    const [riskLevel, setRiskLevel] = useState(initialRiskLevel);
    const [paymentAmount, setPaymentAmount] = useState(299.00); // Gold Tier Price Mockup

        e.preventDefault();
        if (!paymentToken || paymentAmount <= 0) {
            setStatusMessage({ type: 'error', message: "⚠️ Please enter a valid payment token and amount." });
            return;
        }

        setIsSubmitting(true);
        setStatusMessage({ type: null, message: 'System Audit Running... Please wait.' });

        } else {

                    <label className="block text-sm font-medium text-gray-300 mb-1">Premium Insurance Coverage (Gold Tier)</label>
                    <p className="text-4xl font-extrabold text-yellow-400">${paymentAmount.toFixed(2)}</p>
                </div>

                        type="text"
                        value={paymentToken}
                        onChange={(e) => setPaymentToken(e.target.value)}
                        className="w-full p-3 bg-gray-700 border border-red-600 rounded text-white focus:ring-2 focus:ring-red-500"
                        required
                        placeholder="XXXX XXXX XXXX 1234 (Test Fail token: fail...)"
                    />
                </div>

                {/* 제출 버튼 */}
