## ⚖️ 1. GDPR (일반 데이터 보호 규정)

### 과징금 구조 (Article 83)

- **1단계 (경미):** 최대 **€10,000,000** 또는 글로벌 연매출 **2%** 중 높은 금액
  - 컨트롤러/프로세서 의무 위반 해당
- **2단계 (중대):** 최대 **€20,000,000** 또는 글로벌 연매출 **4%** 중 높은 금액
  - 핵심 원칙, 정보주체 권리, 국경 간 전송 위반 해당

### 리스크 산정 공식

```
QLoss_GDPR = max(20,000,000, Turnover × 0.04) × R_GDPR
```

여기서 리스크 노출 계수 `R_GDPR`은:

```
R_GDPR = 0.45 × Defect_Index + 0.35 × Data_Sensitivity + 0.20 × Lack_of_DPO
```

| 변수 | 범위 | 설명 |
|------|------|------|
| `Defect_Index` | 0 ~ 1.0 | 보안 조치 부재 정도 (암호화 미적용, 감사 추적 부재 등) |
| `Data_Sensitivity` | 0 ~ 1.0 | 처리 데이터 민감도 (일반 PII: 0.2, 생체·범죄·정치 의견: 1.0) |
| `Lack_of_DPO` | 0 또는 1 | 데이터보호 책임자(DPO) 미지정 여부 |

### Connect AI 적용 시나리오

| 시나리오 | Defect_Index | Data_Sensitivity | Lack_of_DPO | R_GDPR | 예상 최대 손실 |
|----------|-------------|------------------|-------------|--------|---------------|
| 현재 상태 (추정) | 0.6 | 0.4 | 1 | 0.61 | €12,200,000 |
| DPO 지정 + 암호화 강화 후 | 0.2 | 0.4 | 0 | 0.23 | €4,600,000 |
| 완전 준수 | 0.0 | 0.2 | 0 | 0.07 | €1,400,000 |

> ⚠️ **즉시 조치 필요:** DPO 지정만으로 R_GDPR을 **0.20 포인트** 감소시킬 수 있음

## 🇺🇸 2. CCPA (캘리포니아 소비자 프라이버시법)

### 과징금 구조 (Section 1798.155)

- **비의도적 위반:** 위반당 최대 **$2,500**
- **의도적 위반 / 시정 불이행:** 위반당 최대 **$7,500**
- **소비자 사적 소송권 (Section 1798.150):** 소비자당 **$100 ~ $750** 또는 실제 손해액 중 높은 금액

### 리스크 산정 공식

```
QLoss_CCPA = N_CA_Consumers × (P_Statutory × I_Intentional + P_Breach × (1 - S_Security))
```

| 변수 | 기본값 | 설명 |
|------|--------|------|
| `N_CA_Consumers` | 추정 필요 | 캘리포니아 거주 데이터 주체 수 |
| `P_Statutory` | $2,500 | 기본 법정 과징금 승수 |
| `I_Intentional` | 1.0 (과실) / 3.0 (고의) | 의도성 가중치 |
| `P_Breach` | $750 | 집단소송 평균 법정 합의금 (인당) |
| `S_Security` | 0 ~ 1.0 | 암호화 및 접근제어 수준 |

### Connect AI 적용 시나리오

| CA 사용자 수 | 의도성 | 보안 수준 | 예상 최대 손실 |
|-------------|--------|----------|---------------|
| 1,000명 | 과실 (1.0) | 0.5 | $2,875,000 |
| 10,000명 | 과실 (1.0) | 0.5 | $28,750,000 |
| 10,000명 | 과실 (1.0) | 0.9 | $25,750,000 |

> 📌 **핵심 포인트:** CA 사용자 수가 직접적으로 리스크에 비례 — 사용자 지역 분석 우선 필요

## 🏥 3. HIPAA (건강보험 이동성 및 책임법)

### 과징금 구조 (4단계 책임 등급)

| 등급 | 유형 | 위반당 과징금 | 연간 상한 |
|------|------|-------------|----------|
| Tier 1 | 인지 불가 | $137 ~ $68,928 | $2,067,813 |
| Tier 2 | 합리적 원인 | $1,379 ~ $68,928 | $2,067,813 |
| Tier 3 | 고의적 방임 (시정) | $13,785 ~ $68,928 | $2,067,813 |
| Tier 4 | 고의적 방임 (미시정) | 최소 $68,928 | $2,067,813 |

### 리스크 산정 공식

```
QLoss_HIPAA = C_Culpability_Tier × V_Records × (1 - F_BAA)
```

| 변수 | 값 | 설명 |
|------|---|------|
| `C_Culpability_Tier` | Tier 1: $100 / Tier 2: $1,000 / Tier 3: $10,000 / Tier 4: $50,000 | 보안 정책 감사 기반 등급 계수 |
| `V_Records` | 노출 건수 | 노출된 보호 건강 정보(PHI) 레코드 수 |
| `F_BAA` | 0 또는 1 | 제3자 벤더와의 사업 제휴 계약(BAA) 존재 여부 |

### Connect AI 적용 판단

| 항목 | 상태 | 비고 |
|------|------|------|
| PHI 데이터 수집 여부 | ❌ 미수집 | 현재 의료 데이터 취급하지 않음 |
| 향후 헬스케어 확장 계획 | ⚠️ 미정 | 확장 시 즉시 적용 대상 |
| BAA 체결 현황 | N/A | 해당 없음 |

> ✅ **현재 상태:** HIPAA 노출도 낮음. 단, 헬스케어 AI 진출 시 사전 BAA 체결 및 PHI 암호화 필수

## 🤖 4. EU AI Act (인공지능법)

### 과징금 구조 (시스템 리스크 분류별)

| 분류 | 해당 조항 | 최대 과징금 |
|------|----------|-----------|
| **금지된 AI 관행** | Article 5 | €35,000,000 또는 글로벌 매출 **7%** |
| **고위험 AI 비준수** | Articles 6-15 | €15,000,000 또는 글로벌 매출 **3%** |
| **허위 정보 제공** | — | €7,500,000 또는 글로벌 매출 **1.5%** |

### 리스크 산정 공식

```
QLoss_AI = System_Class_Limit × (1 - (Data_Governance_Score + Human_Oversight_Score) / 2)
```

| 변수 | 범위/값 | 설명 |
|------|--------|------|
| `System_Class_Limit` | 금지: €35M / 고위험: €15M / GPAI: €7.5M | 시스템 분류별 과징금 한도 |
| `Data_Governance_Score` | 0 ~ 1.0 | 학습 데이터 품질, 편향 감소, 문서화 수준 |
| `Human_Oversight_Score` | 0 ~ 1.0 | 비상 중단 장치, 실시간 킬 스위치, 감사 대시보드 |

### Connect AI 적용 시나리오

Connect AI는 **범용 AI(GPAI)** 시스템으로 분류될 가능성이 높으며, 일부 기능은 **고위험 AI**로 재분류될 수 있습니다.

| 시나리오 | 시스템 분류 | Data Gov. | Human Oversight | 예상 최대 손실 |
|----------|-----------|-----------|-----------------|---------------|
| 현재 상태 (추정) | GPAI | 0.3 | 0.2 | €5,625,000 |
| 거버넌스 강화 후 | GPAI | 0.7 | 0.6 | €2,625,000 |
| 고위험 분류 시 | High-Risk | 0.3 | 0.2 | €11,250,000 |
| 완전 준수 | GPAI | 0.9 | 0.9 | €750,000 |

> 🚨 **최우선 조치:** 고위험 AI 재분류 방지를 위한 투명성 보고서 작성 + 킬 스위치 구현
